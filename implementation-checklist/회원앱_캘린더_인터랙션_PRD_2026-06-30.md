## 제품에서 달력이 맡는 일

Bubli의 달력은 독립적인 날짜표가 아니다. 프리랜서가 오늘 해야 할 일, 프로젝트룸 WBS 마감, TODO 마감, 외부 Google Calendar 일정을 한 화면에서 비교하는 업무 기준 화면이다.

- `roomId`가 없는 일정은 개인 일정이다.
- `roomId`가 있는 일정은 현재 고른 프로젝트룸 일정이다.
- `taskId`, `wbsItemId`가 있으면 TODO 또는 WBS와 연결된 일정이다.
- `googleEventId`, `syncStatus`가 있으면 Google Calendar 연동 상태를 표시한다.
- 대시보드와 데스크탑 일정 버블은 같은 서버 원본을 본다.

## API 기준

최신 API 명세는 `/api/schedules`가 아니라 아래 구글 캘린더 경유 계약을 기준으로 한다.

- `GET /api/calendar/google/connect`
- `GET /api/calendar/google/callback`
- `GET /api/calendar/events?start=&end=&roomId=`
- `POST /api/calendar/events`
- `PATCH /api/calendar/events/{id}`
- `DELETE /api/calendar/events/{id}`

프론트는 Google Calendar를 직접 호출하지 않는다. 항상 Bubli 백엔드 API를 거친다.

## 살릴 인터랙션

- 주간 날짜 버튼: 날짜를 누르면 오른쪽이 아니라 같은 화면의 하루 타임라인이 바뀐다.
- 하루 타임라인: 일정은 시간순으로 보이고, 상태는 완료, 진행 중, 대기, 오류로 나뉜다.
- 반복 일정 작성: 반복 토글, 매일/매주/매월 선택, 요일 칩 선택을 제공한다.
- Google Calendar 연결: 프론트는 연결 시작 URL만 열고, 토큰과 동기화는 백엔드가 맡는다.
- 충돌 확인: `syncStatus`가 실패면 자동 확정하지 않고 확인 필요로 남긴다.

## 현재 구현 주의점

v15 API의 `CalendarEventRequest`에는 반복 규칙 필드가 없다. 그래서 반복 UI는 화면 상호작용과 일정 후보 패턴으로 유지하고, 실제 저장 요청에는 현재 명세의 단건 일정 필드만 보낸다.

반복 규칙이 API에 추가되면 현재 화면 상태를 그대로 `repeatRule` 또는 서버가 정한 필드로 연결하면 된다.
