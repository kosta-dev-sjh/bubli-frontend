# Bubli 회원 웹 앱 화면 책임 체크리스트

이 문서는 회원 웹 앱을 다시 손볼 때 빠뜨리거나 중복하지 않기 위한 작업 기준이다. 공개 사이트는 소개와 다운로드를 맡고, 회원 웹 앱은 실제 업무를 처리하는 공간이다.

## 공통 원칙

- 회원 웹 앱은 랜딩페이지가 아니다. 소개 문구, 홍보 문구, "이런 기능이 있습니다" 식 설명을 화면 중심에 두지 않는다.
- 화면에는 사용자가 지금 조작할 수 있는 것만 둔다. 의미 없는 새로고침, 중복 이동 버튼, 내부 상태 확인용 문구는 빼거나 복구 상태에만 둔다.
- 프로젝트룸을 선택하면 세션 전체가 그 프로젝트룸 맥락을 유지한다. 프로젝트룸 화면으로 잠깐 이동하고 끝나는 구조가 아니다.
- 개인 대시보드와 개인 자료는 언제나 따라오는 개인 영역이다. 프로젝트룸 선택 목록 안에 개인 대시보드와 개인 자료를 섞지 않는다.
- 개발 확인용 데이터가 보일 수는 있지만, 사용자 화면에는 "목업", "가짜", "확인용", "서버 연결 전 화면 확인" 같은 말이 나오면 안 된다.
- `docs/visual-target/모션&UI 참고용.md`는 인터랙션 참고 자료다. 코드를 그대로 베끼는 기준이 아니라, Bubli 화면 책임에 맞게 재해석한다.
- 디자인보드는 구현 대상이 아니다. 색, 밀도, 모션, 위젯 느낌을 참고하되 실제 화면 책임은 프론트 명세와 이 체크리스트를 따른다.

## 반복 UI 요구

- 비회원 페이지와 회원 앱의 톤은 이어져야 한다. 다만 회원 앱은 더 조용하고 밀도 있는 작업 도구여야 한다.
- 큰 제목, 큰 카드, 장식 버블을 남발하지 않는다. 강조는 필요한 순간에만 쓴다.
- 버블은 브랜드 컨셉이지만, 회원 웹 앱 안에서 장식 아이콘처럼 쓰지 않는다. Tauri 데스크탑 위젯과 기능 신호에만 쓴다.
- 청록, 민트, aqua, teal 계열은 피한다.
- 라이트 모드는 맑고 얇게, 다크 모드는 남색 박스가 아니라 얇은 rim과 inner highlight 중심으로 둔다.
- 버튼과 메뉴는 말랑하게 만들되, 의미 없는 구슬 장식은 넣지 않는다.
- 메뉴 아이콘에는 hover tooltip이 있어야 한다. 아이콘만 남겨도 어디로 가는지 알 수 있어야 한다.
- 새로고침 버튼은 기본 액션이 아니다. 자동 갱신이 기본이고, 실패 상태에서만 "다시 연결"을 둔다.
- 한 화면에 같은 의미의 위치 표시, 현재 메뉴, 상단 칩을 중복으로 늘어놓지 않는다.

## 화면별 책임

| 화면 | 들어갈 것 | 빼야 할 것 | 현재 보완 기준 |
|---|---|---|---|
| `/app` 대시보드 | 개인 홈, 카드 추가/제거/재배치, 선택한 프로젝트룸에서 끌어온 카드, 오늘 할 일/일정/확인 필요/타이머/최근 자료 | 소개 문구, 새로고침, 자료보드/프로젝트룸 이동 버튼 반복 | 편집 모드에서 드래그로 순서 변경과 제거를 명확히 보이게 한다 |
| `/app/project-rooms` | 프로젝트룸 목록, 생성, 선택, 현재 룸 전환 | 개인 자료/개인 대시보드 항목 섞기, 중복 "프로젝트룸" 메뉴 | 프로젝트룸을 고르면 전체 세션 맥락이 유지되어야 한다 |
| `/app/project-rooms/[roomId]` | 룸 홈, 작업 요약, 자료, 일정, 대화, 멤버, 후보 확인 진입 | 랜딩식 설명, 빈 카드 나열 | 선택한 룸에서 해야 할 일을 바로 파악하게 한다 |
| `/app/resources` 개인 자료 | 로컬 폴더 연결 상태, 개인 색인, 최근 동기화, 미리보기 | 업로드, 드래그앤드롭, 공용 공유 표현 | 개인 자료는 프로젝트룸을 바꿔도 따라오는 내 작업 자료다 |
| `/app/project-rooms/[roomId]/resources` 룸 자료 | 공용 자료 업로드, 드래그앤드롭, 멤버 공용 미리보기, 승인 흐름 | 로컬 폴더 동기화 | 룸 자료는 프로젝트룸 멤버가 함께 보는 자료다 |
| `/app/project-rooms/[roomId]/work` 작업판 | WBS 보기, 칸반 보기, TODO 상태 변경, 후보 승인 진입 | WBS/칸반/후보를 한 줄에 억지로 압축, 과한 설명 | WBS와 칸반은 전환해서 보는 구조를 우선한다 |
| `/app/calendar` 일정 | 월/주 보기, 개인 일정, 프로젝트룸 일정, 반복 일정, Google Calendar 연결 | 기본 새로고침, 구글 버튼만 덩그러니 두기 | 구글 캘린더처럼 스캔 가능한 월/주 구조를 둔다 |
| `/app/chat` 대화 | 프로젝트룸 대화, 1:1 대화, 자료 연결, 보이스 시작점, 에이전트 호출 흐름 | 내가 입력 중인데 상대 입력처럼 보이는 표시, 친구 기능 누락 | 대화 목록에서 룸/1:1을 구분하고 보이스 진입을 노출한다 |
| `/app/agent` 후보 | 개인 후보와 룸 후보 분리, 승인/보류/수정 | 확정 데이터처럼 보이는 후보 | 후보는 승인 전 제안으로만 보인다 |
| `/app/settings` 설정 | 계정, 언어, 알림, 개인정보 동의, 로컬 폴더, Tauri 동기화, 백업/복구 | "설정을 불러오지 못했습니다"만 있는 빈 화면 | API 실패해도 설정 영역 구조는 보여야 한다 |
| `/app/desktop/widgets` | Tauri 위젯 검토 화면 | 실제 위젯 창처럼 오해되는 웹앱 카드 | 실제 데스크탑 위젯은 `/desktop-widget`와 Tauri 창에서 검증한다 |

## 자료보드 경계

- 개인 자료: 로컬 폴더 동기화로 들어온다. 데스크탑 앱에서 폴더를 지정한다. 브라우저 업로드를 두지 않는다.
- 프로젝트룸 자료: 업로드와 드래그앤드롭으로 들어온다. 멤버가 함께 보는 공용 자료다. 로컬 폴더 동기화가 아니다.
- 개인 자료를 프로젝트룸에 공유할 때는 승인 흐름을 거친다. 개인 자료와 룸 자료를 한 목록처럼 섞지 않는다.

## 대화 경계

- 프로젝트룸 대화와 1:1 대화는 서버 메시지가 원본이다.
- 보이스는 프로젝트룸 또는 1:1 맥락에서 시작한다.
- 내가 입력하는 동안 상대가 입력 중인 것처럼 보이는 UI는 두지 않는다.
- 에이전트 호출은 대화 안에서 가능하지만, 확정 데이터로 바로 쓰지 않는다.

## 일정 경계

- 개인 일정과 프로젝트룸 일정은 한 화면에서 볼 수 있지만, 출처는 표시한다.
- 반복 일정은 사용자가 알아볼 수 있게 주기와 요일을 드러낸다.
- Google Calendar 연결은 설정/상태 카드로 둔다. 헤더에 덩그러니 놓지 않는다.
- 수동 새로고침은 실패 상태의 다시 연결에만 둔다.

## 작업판 경계

- WBS는 문서에서 확정된 작업 구조다.
- TODO/칸반은 승인된 작업의 실행 상태다.
- 후보는 에이전트가 만든 제안이다. WBS/TODO와 같은 확정 영역에 섞지 않는다.
- WBS와 칸반은 서로 연결되지만, 한 줄에 다 밀어 넣지 않는다. 전환 또는 분리 레이아웃을 쓴다.
