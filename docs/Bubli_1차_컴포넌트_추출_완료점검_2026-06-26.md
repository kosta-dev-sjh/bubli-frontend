# Bubli 1차 컴포넌트 추출 P0 점검

작성일: 2026-06-26  
기준: v15 기획/API, 디자인보드 v20, `Bubli_프론트엔드_개발계획_AtoZ_2026-06-26.md`

## 결론

1차 컴포넌트 추출은 **완료가 아니라 P0 진행 중**으로 본다.

지금 필요한 일은 디자인보드 v20 기준 재조립이 아니라, AtoZ의 P0 기준대로 **부품 인벤토리를 확정하고 누락 5종 골격을 먼저 채우는 것**이다. 디자인 전면 개편은 P0/P1 뒤에 진행한다.

## P0 완료 기준

1차 컴포넌트 추출은 아래 기준을 모두 만족해야 완료로 본다.

| 기준 | 현재 판단 |
|---|---|
| v15 핵심 화면이 라우트에 존재한다 | 부분 충족. `/app/calendar`와 `/app/agent-suggestions`는 추가했고, 나머지 보강 3종이 남아 있다. |
| 프로젝트룸, 자료보드, 대시보드, 작업판, 소통, 설정, 버블이 분리되어 있다 | 충족 |
| 게스트, 이메일/비밀번호 회원가입, 프로젝트/프로젝트룸 분리 같은 삭제된 흐름이 새 라우트 기준에 없다 | 충족 |
| 핵심 기능마다 Storybook에서 확인할 대표 컴포넌트가 있다 | 부분 충족. 누락 5종의 4상태 story가 필요하다. |
| API 계약이 확정되기 전에도 화면을 확인할 수 있다 | 충족 |
| 디자인보드 v20 전면개편 때 살릴 구조와 버릴 점검용 패널을 구분할 수 있다 | 부분 충족. `component-inventory.csv`를 계속 갱신한다. |

## 지금 하지 않을 일

- 디자인보드 v20 기준으로 실제 화면을 세부 재조립하지 않는다.
- 정책, 경계, API 설명 패널을 삭제하지 않는다.
- API 계약이 확정되기 전에 화면 전체를 실제 데이터 중심으로 묶지 않는다.
- 세부 여백, 색, 질감 polish를 컴포넌트별로 따로 고치지 않는다.
