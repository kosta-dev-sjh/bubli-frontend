# Tauri 데스크탑 앱 명세

이 문서는 Tauri 앱에 들어갈 기능과 웹 앱에 남겨야 할 기능을 나눈다.

## 한 줄 정의

Tauri 앱은 Bubli 회원 웹 앱을 데스크탑에서 실행하고, 개인 버블 위젯과 로컬 기능을 붙이는 껍데기다.

새 업무 서비스를 따로 만드는 것이 아니다.

## 앱 구조

```text
Tauri 데스크탑 앱
├─ 메인 창
│  └─ WebView로 회원 웹 앱 /app 표시
├─ 위젯 창
│  ├─ 항상 위 표시
│  ├─ 위치 이동
│  ├─ 투명도 모드
│  ├─ 최소화 모드
│  └─ 버블별 상호작용
├─ 로컬 기능
│  ├─ 개인 관리 폴더 선택
│  ├─ 파일 변경 감지
│  ├─ SQLite 캐시
│  ├─ 개인 에이전트 원문 대화
│  ├─ 위젯 사용 이벤트
│  └─ 타이머 복구
└─ 시스템 기능
   ├─ 트레이
   ├─ 앱 시작/종료
   ├─ 알림
   └─ 활동 감지 동의
```

## 메인 창

메인 창은 회원 웹 앱을 표시한다.

| 항목 | 기준 |
|---|---|
| 표시 화면 | `/app` 회원 웹 앱 |
| 로그인 | 구글 OAuth 기준. Tauri는 refresh token을 운영체제 보안 저장소에 저장 |
| API 호출 | 웹 앱과 같은 API 서버 호출 |
| 에이전트 호출 | Tauri가 직접 모델 또는 에이전트 서버를 호출하지 않음 |
| 화면 구성 | 웹 앱의 사이드바, 상단바, 본문 구조를 그대로 사용 |

## 위젯 창

위젯은 개인 영역이다. 프로젝트룸 화면을 그대로 줄여 넣지 않는다.
Tauri 위젯은 20v 디자인보드에 있던 기능 구성을 기준으로 잡는다.
20v는 단순한 시각 참고가 아니라 기능 체크리스트다.
20v에 있던 기능은 같은 책임으로 가져오고, 현재 비어 있는 부분만 채운다.
현재 코드나 화면에서 빠져 보이는 기능은 삭제된 기능이 아니라 구현할 누락 항목으로 본다.
나머지 회원앱 화면과 세부 디자인은 단계적으로 손보더라도, Tauri 위젯 기능 범위는 20v보다 작아지면 안 된다.
단, 기획에서 삭제된 받을돈/페이 위젯은 되살리지 않는다.

| 버블 | 보여줄 정보 | 원본 |
|---|---|---|
| TODO 버블 | 오늘 내 TODO, 선택 프로젝트룸 TODO 요약 | 서버 TODO, 로컬 캐시 |
| 일정 버블 | 오늘 일정, 마감일, 구글 캘린더 연결 항목 | 서버 일정, 캘린더 동기화 결과 |
| 타이머 버블 | 현재 작업 시간, 일시정지, 재개, 종료 | 서버 `time_logs`, Tauri `local_timer_state` |
| 자료 제안 버블 | 새 자료, 분석 완료, 확인 필요 항목 | 서버 resources, agent_suggestions |
| 메모 버블 | 개인 메모, 작업 중 메모 | Tauri 로컬 또는 개인 API |
| 소통 알림 버블 | 프로젝트룸 채팅, 댓글, 자료 버전 알림 | 서버 notifications, chat_messages |
| 에이전트 버블 | 승인 대기 후보, 하루정리 후보 | 서버 agent_jobs, agent_suggestions, 로컬 요약 |
| 알림 버블 | 오늘 놓치면 안 되는 상태 변화와 확인 필요 항목 | 서버 notifications, 로컬 표시 캐시 |

## 위젯 상태

| 상태 | 설명 | 구현 기준 |
|---|---|---|
| 기본 | 정보가 또렷하게 보이는 일반 모드 | 작업 중 빠르게 확인 |
| 반투명 | 배경 작업을 덜 가리는 모드 | 텍스트 대비 유지 |
| 고스트 | 화면 위에 오래 띄워도 방해가 적은 모드 | 핵심 숫자와 아이콘만 강조 |
| 최소화 | 작은 바 또는 orb로 접은 상태 | 알림 수, 타이머, 현재 프로젝트룸 정도만 표시 |

4상태 비교 UI는 구현 검토용이다. 실제 Tauri 위젯은 사용자가 상태를 전환하면서 본다.

## 20v 기능 동등성 기준

Tauri 위젯 작업은 새 디자인으로 줄여서 다시 만드는 일이 아니다.
20v에 있던 기능을 같은 책임으로 가져오고, 비어 있는 부분만 채운다.

| 기준 | 반드시 남길 내용 |
|---|---|
| 버블 단위 | TODO, 에이전트, 소통, 타이머, 메모, 일정, 자료, 알림 |
| 표시 상태 | 기본, 반투명, 고스트, 최소화 |
| 항목 동작 | 확인, 고정, 숨김, 세부 열기 |
| 창 동작 | 드래그 이동, 위치 기억, always-on-top, 프레임리스 창 |
| 고스트 동작 | 낮은 시각 강조, 클릭 투과 |
| 최소화 동작 | Bubble Bar 또는 Dock Orb |
| 시스템 동작 | 트레이, 전역 단축키, 앱 재시작 후 상태 복구 |
| 데이터 동작 | 서버 원본 조회, Tauri SQLite 표시 캐시, 로컬 사용 이벤트, 집계 후 서버 반영 |

`/app/desktop/widgets`는 이 기능들을 설명하는 회원앱 안의 검토 화면이다.
실제 떠 있는 버블 창은 Tauri window로 만든다.

## Tauri IPC

Tauri IPC는 로컬 기기 기능만 맡는다.

| IPC | 설명 | 서버 반영 |
|---|---|---|
| `select_managed_folder` | 사용자가 폴더를 직접 선택 | 없음 |
| `scan_managed_folder` | 선택 폴더 파일 메타데이터 확인 | 필요 시 sync API |
| `watch_managed_folder` | 앱 실행 중 파일 변경 감지 | 필요 시 sync API |
| `search_local_files` | 로컬 SQLite/FTS 기준 검색 | 없음 |
| `sync_room_messages` | 서버 채팅을 로컬 캐시에 반영 | 서버 조회 |
| `backup_local_sqlite` | 로컬 SQLite 백업 생성 | 없음 |
| `read_activity_context` | 동의 후 앱 이름, 창 제목, 머문 시간 확인 | 사용자가 동의한 범위만 API 반영 |
| `record_widget_usage_event` | 버블 사용 상세 이벤트 기록 | 로컬 저장 |
| `rollup_widget_usage` | 상세 이벤트를 날짜별 집계로 압축 | 집계 후 API 반영 |
| `recover_timer_state` | 앱 재실행 시 타이머 상태 비교 | 필요 시 time_logs API |
| `get_widget_window_state` | 현재 위젯 창 상태 확인 | 없음 |
| `set_widget_window_mode` | 기본, 반투명, 고스트, 최소화 상태 전환 | 없음 |
| `set_widget_window_position` | 사용자가 옮긴 위젯 위치 저장 | 없음 |
| `set_widget_always_on_top` | 위젯을 화면 위에 고정 | 없음 |
| `set_widget_click_through` | 고스트 상태에서 클릭 투과 전환 | 없음 |
| `register_widget_shortcut` | 전역 단축키 등록 | 없음 |
| `toggle_widget_dock_orb` | 최소화 orb 또는 bar 전환 | 없음 |
| `update_widget_tray_state` | 트레이 표시 상태 갱신 | 없음 |

## 저장 기준

| 데이터 | 저장 위치 | 이유 |
|---|---|---|
| 프로젝트룸, 자료, WBS, TODO, 일정, 채팅 원본 | 서버 DB | 웹과 앱이 같은 원본을 봐야 함 |
| 업로드 파일 | S3와 DB 메타데이터 | 권한 검증과 공유 범위 관리 |
| 프로젝트룸 채팅 캐시 | Tauri SQLite | 빠른 표시와 오프라인 복구 |
| 개인 에이전트 원문 대화 | Tauri SQLite | 개인 로컬 맥락 보호 |
| 위젯 상세 사용 이벤트 | Tauri SQLite | 사용자의 세부 사용 패턴을 서버에 과하게 올리지 않기 위함 |
| 위젯 일별 집계 | 로컬 집계 후 서버 저장 | 하루정리와 여러 기기 조회에 필요 |
| 타이머 원본 | 서버 `time_logs` | 작업시간은 웹과 앱에서 다시 보여야 함 |
| 타이머 복구 상태 | Tauri `local_timer_state`, `local_sync_outbox` | 비정상 종료 복구 |

## 활동 감지 경계

활동 감지는 사용자가 켠 경우에만 동작한다.

수집 가능한 값은 아래 정도로 제한한다.

- 앱 이름
- 창 제목
- 머문 시간
- 사용자가 연결한 프로젝트룸 후보

수집하지 않는 값은 아래와 같다.

- 화면 전체 내용
- 키보드 입력
- 브라우저 본문 전체
- 개인 파일 내용의 자동 업로드

## Tauri 구현 완료 기준

| 항목 | 완료 기준 |
|---|---|
| WebView | `/app`이 로그인 후 정상 표시된다 |
| 위젯 | 기본, 반투명, 고스트, 최소화 상태가 동작한다 |
| 폴더 | 사용자가 선택한 폴더만 감지한다 |
| SQLite | 채팅 캐시, 위젯 캐시, 타이머 복구 상태가 저장된다 |
| 동기화 | 서버 반영 대기열이 중복 요청을 막는다 |
| 보안 | refresh token은 localStorage나 SQLite 평문에 저장하지 않는다 |
| 권한 | 활동 감지와 폴더 접근은 사용자가 직접 켜야 한다 |

한마디로, Tauri는 웹 앱을 앱처럼 띄우고 개인 로컬 기능을 붙이는 역할이다.
