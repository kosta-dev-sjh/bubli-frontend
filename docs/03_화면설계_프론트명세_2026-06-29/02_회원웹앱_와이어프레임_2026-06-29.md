# 회원 웹 앱 와이어프레임

이 문서는 회원 웹 앱의 낮은 충실도 화면 설계다. 색과 장식보다 정보 순서와 행동 흐름을 먼저 본다.

## 공통 레이아웃

회원 웹 앱은 업무 도구다. 마케팅 페이지처럼 큰 히어로를 반복하지 않는다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 상단바: 현재 위치 / 검색 / 알림 / 프로필                       │
├───────────────┬──────────────────────────────────────────────┤
│ 사이드바       │ 본문                                          │
│ - 대시보드     │ 화면별 주요 정보                              │
│ - 프로젝트룸   │ 작업 버튼                                     │
│ - 자료보드     │ 목록, 상세, 검토 패널                          │
│ - 작업판       │                                              │
│ - 채널         │                                              │
│ - 일정         │                                              │
│ - 설정         │                                              │
└───────────────┴──────────────────────────────────────────────┘
```

공통 사이드바는 회원 앱 안에서만 쓴다. 공개 사이트에는 쓰지 않는다.

## `/app` 대시보드

대시보드는 사용자를 따라다니는 개인 화면이다. 여러 프로젝트룸에서 내가 맡은 일을 모아 보여준다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 오늘 할 일                                                    │
│ [프로젝트룸 선택 필터] [오늘] [이번 주] [정렬]                 │
├──────────────────────┬──────────────────────┬────────────────┤
│ 내 TODO              │ 확인 필요 항목        │ 일정            │
│ - 마감 임박          │ - 계약서 누락값       │ - 오늘 미팅      │
│ - 진행 중            │ - 요구사항 충돌       │ - 납품일         │
├──────────────────────┴──────────────────────┴────────────────┤
│ 진행 중 타이머 / 최근 자료 / 에이전트 제안 요약                │
└──────────────────────────────────────────────────────────────┘
```

대시보드에 프로젝트룸 공용 정보를 전부 넣지 않는다. 프로젝트룸 상세 정보는 프로젝트룸 안에서 본다.

## `/app/project-rooms`

프로젝트룸 목록은 사용자가 업무 단위를 고르는 화면이다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 프로젝트룸                                                    │
│ [새 프로젝트룸] [검색] [상태 필터]                            │
├──────────────────────────────────────────────────────────────┤
│ 프로젝트룸 카드 목록                                           │
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐            │
│ │ 룸 이름       │ │ 룸 이름       │ │ 룸 이름       │            │
│ │ 내 TODO 3    │ │ 확인 필요 1  │ │ 일정 2       │            │
│ │ 최근 자료     │ │ 최근 채팅     │ │ 진행률        │            │
│ └──────────────┘ └──────────────┘ └──────────────┘            │
└──────────────────────────────────────────────────────────────┘
```

프로젝트와 프로젝트룸을 따로 만들지 않는다. 사용자가 만드는 업무 단위는 프로젝트룸 하나다.

## `/app/project-rooms/{roomId}`

프로젝트룸 상세는 프로젝트 단위의 업무 맥락을 모아 보는 화면이다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 프로젝트룸 이름 / 클라이언트 / 상태 / 멤버                     │
│ [자료 업로드] [친구 초대] [설정]                               │
├──────────────────────┬──────────────────────┬────────────────┤
│ 최근 자료             │ WBS/TODO 요약         │ 채팅/알림       │
│ 계약서, 회의록        │ 승인 대기 후보        │ 최근 결정       │
├──────────────────────┴──────────────────────┴────────────────┤
│ 에이전트 제안: 확인 필요 항목, WBS 후보, TODO 후보             │
└──────────────────────────────────────────────────────────────┘
```

초대 버튼은 친구 목록에서 선택하는 흐름으로만 연결한다.

## `/app/resources`

자료보드는 받은 자료를 업무 기준으로 바꾸는 핵심 화면이다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 자료보드                                                      │
│ [개인 자료] [프로젝트룸 자료] [업로드] [검색]                  │
├──────────────────────┬───────────────────────────────────────┤
│ 자료 목록             │ 자료 상세                              │
│ - 계약서              │ 파일 정보 / 버전 / 댓글                 │
│ - 요구사항            │ 에이전트 분석 결과                      │
│ - 회의록              │ 확인 필요 항목                          │
│                       │ [후보 승인] [수정] [보류]               │
└──────────────────────┴───────────────────────────────────────┘
```

개인 자료와 프로젝트룸 자료를 섞어 보이게 만들면 안 된다. 개인 자료는 사용자가 공유를 승인해야 프로젝트룸 자료가 된다.

## `/app/project-rooms/{roomId}/work`

WBS/작업판은 승인된 후보가 실제 업무가 되는 화면이다.

```text
┌──────────────────────────────────────────────────────────────┐
│ WBS / 작업판                                                   │
│ [WBS 보기] [칸반 보기] [담당자] [상태]                         │
├──────────────────────┬───────────────────────────────────────┤
│ WBS 트리              │ 작업 상세                              │
│ 1. 분석               │ 제목, 설명, 담당자, 마감일              │
│ 1-1. 자료 확인        │ 연결 자료, 연결 후보, 댓글              │
│ 1-2. 화면 제작        │ [상태 변경] [일정 연결]                 │
└──────────────────────┴───────────────────────────────────────┘
```

에이전트가 만든 것은 후보이고, 사용자가 승인해야 실제 WBS/TODO가 된다.

## `/app/project-rooms/{roomId}/chat`

채팅은 자료와 작업 맥락을 잃지 않게 프로젝트룸 안에 둔다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 프로젝트룸 채팅                                                │
│ [텍스트] [보이스 시작] [자료 연결] [에이전트 호출]              │
├──────────────────────┬───────────────────────────────────────┤
│ 채널/참여자           │ 메시지 목록                             │
│ 프로젝트룸 채팅       │ 결정사항 표시                            │
│ 1:1 채팅              │ 자료 링크, TODO 링크                     │
└──────────────────────┴───────────────────────────────────────┘
```

비회원 게스트 채팅과 게스트 보이스 참여는 만들지 않는다.

## `/app/desktop/widgets`

이 라우트는 Tauri 위젯을 설명하거나 QA하기 위한 회원 앱 안의 미리보기 화면이다. 실제 데스크탑 위젯 창과 완전히 같은 것은 아니다.

```text
┌──────────────────────────────────────────────────────────────┐
│ 데스크탑 버블 미리보기                                         │
│ [기본] [반투명] [고스트] [최소화] [라이트/다크]                 │
├──────────────────────────────────────────────────────────────┤
│ 데스크탑 캔버스 또는 4상태 비교 영역                           │
│ - TODO 버블                                                    │
│ - 일정 버블                                                    │
│ - 타이머 버블                                                  │
│ - 자료 제안 버블                                               │
│ - 알림 버블                                                    │
└──────────────────────────────────────────────────────────────┘
```

위젯 미리보기는 디자인 검증용이다. 실제 데이터 저장과 로컬 권한은 Tauri 명세를 따른다.

## 화면별 완료 기준

| 화면 | 최소 완료 기준 |
|---|---|
| 대시보드 | 내 TODO, 일정, 확인 필요 항목, 진행 중 타이머가 사용자 기준으로 구분된다 |
| 프로젝트룸 목록 | 프로젝트룸 생성, 목록, 상태, 최근 활동이 보인다 |
| 자료보드 | 개인 자료와 프로젝트룸 자료가 분리되고, 후보 승인 흐름이 보인다 |
| WBS/작업판 | 후보와 확정 작업이 구분되고, 담당자와 상태가 보인다 |
| 채팅 | 프로젝트룸 채팅과 1:1 채팅의 위치가 구분된다 |
| 위젯 미리보기 | 기본, 반투명, 고스트, 최소화 상태가 한 화면에서 비교된다 |

한마디로, 회원 웹 앱의 화면은 자료가 작업으로 바뀌는 순서를 보여줘야 한다.
