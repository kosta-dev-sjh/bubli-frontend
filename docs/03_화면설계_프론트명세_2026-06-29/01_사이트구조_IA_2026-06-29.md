# Bubli 사이트 구조와 IA

이 문서는 Bubli의 공개 사이트, 회원 웹 앱, Tauri 데스크탑 앱이 각각 무엇을 맡는지 정리한다.

## 전체 구조

```text
Bubli
├─ 공개 사이트
│  ├─ /
│  ├─ /features
│  ├─ /download
│  ├─ /faq
│  └─ /login
├─ 회원 웹 앱
│  ├─ /app
│  ├─ /app/project-rooms
│  ├─ /app/project-rooms/{roomId}
│  ├─ /app/project-rooms/{roomId}/resources
│  ├─ /app/project-rooms/{roomId}/work
│  ├─ /app/project-rooms/{roomId}/chat
│  ├─ /app/resources
│  ├─ /app/schedule
│  ├─ /app/agent
│  └─ /app/settings
└─ Tauri 데스크탑 앱
   ├─ WebView: /app
   ├─ 데스크탑 버블 위젯
   ├─ 개인 관리 폴더
   ├─ SQLite 캐시
   ├─ 트레이와 상단 고정
   └─ 활동 감지
```

## 영역별 역할

| 영역 | 목적 | 로그인 | 주요 사용자 행동 |
|---|---|---|---|
| 공개 사이트 | 서비스 이해와 앱 다운로드 | 필요 없음 | 기능 확인, 앱 다운로드, 구글 로그인 진입 |
| 회원 웹 앱 | 실제 업무 관리 | 필요 | 프로젝트룸 생성, 자료 정리, 작업 확인, 채팅 |
| Tauri 앱 | 웹 앱에 로컬 기능 추가 | 필요 | 버블 확인, 폴더 감지, 로컬 캐시, 앱 활동 기록 |

## 공개 사이트 IA

공개 사이트는 제품 소개와 설치 진입만 맡는다.

```text
/
├─ 핵심 문장: 받은 자료를, 오늘 할 일로
├─ 문제: 자료, 일정, 대화, TODO가 흩어짐
├─ 해결: 프로젝트룸, 자료보드, 대시보드, 버블
├─ 주요 화면 미리보기
├─ 데스크탑 앱 다운로드
└─ 구글 로그인 진입
```

공개 사이트에는 업무 데이터가 나오지 않는다. 비회원은 프로젝트룸, 채팅, 보이스, 자료보드에 들어가지 않는다.

## 회원 웹 앱 IA

회원 웹 앱은 업무를 자세히 정리하는 곳이다.

```text
/app
├─ 대시보드
│  ├─ 오늘 내 TODO
│  ├─ 일정
│  ├─ 확인 필요 항목
│  ├─ 진행 중 타이머
│  └─ 에이전트 제안 요약
├─ 프로젝트룸
│  ├─ 프로젝트룸 목록
│  ├─ 프로젝트룸 상세
│  ├─ 프로젝트룸 자료보드
│  ├─ WBS/작업판
│  ├─ 채팅/보이스
│  ├─ 멤버/친구 초대
│  └─ 설정
├─ 개인 자료보드
│  ├─ 개인 자료 목록
│  ├─ 자료 상세
│  ├─ 프로젝트룸 공유 승인
│  └─ 에이전트 분석 결과
├─ 일정
├─ 에이전트 제안함
└─ 설정
```

대시보드는 사용자 기준이다. 프로젝트룸 하나의 상세 현황판이 아니다.

자료보드는 개인 자료와 프로젝트룸 자료를 구분한다. 개인 자료는 사용자가 공유하기 전까지 프로젝트룸 멤버에게 보이지 않는다.

## Tauri 앱 IA

Tauri 앱은 회원 웹 앱을 새로 만들지 않는다. 같은 회원 웹 앱을 WebView로 열고, 데스크탑에서만 가능한 기능을 붙인다.

```text
Tauri 앱
├─ 메인 창
│  └─ WebView로 /app 표시
├─ 버블 위젯 창
│  ├─ TODO 버블
│  ├─ 일정 버블
│  ├─ 타이머 버블
│  ├─ 자료 제안 버블
│  ├─ 메모 버블
│  └─ 소통 알림 버블
├─ 로컬 기능
│  ├─ 개인 관리 폴더 선택
│  ├─ 파일 변경 감지
│  ├─ SQLite 캐시
│  ├─ 개인 에이전트 원문 대화
│  └─ 타이머 복구 대기열
└─ 시스템 기능
   ├─ 트레이
   ├─ 상단 고정
   ├─ 투명도 조절
   └─ 활동 감지 동의
```

Tauri 기능은 사용자의 기기 안에서 동작하는 개인 기능이다. 프로젝트룸 원본 데이터는 서버 API 기준으로 관리한다.

## 구현 우선순위

| 우선순위 | 화면 | 이유 |
|---:|---|---|
| 1 | `/app` 대시보드 | 여러 프로젝트룸의 오늘 할 일을 바로 보여줘야 한다 |
| 1 | `/app/project-rooms` | 업무 단위인 프로젝트룸을 만들고 들어가는 시작점이다 |
| 1 | `/app/resources` | 받은 자료를 업무 후보로 바꾸는 핵심 입구다 |
| 1 | `/app/project-rooms/{roomId}/work` | 승인된 후보가 실제 WBS/TODO가 되는 곳이다 |
| 2 | `/app/project-rooms/{roomId}/chat` | 자료와 대화가 흩어지지 않게 같은 맥락에 둔다 |
| 2 | `/app/desktop/widgets` 또는 Tauri 위젯 미리보기 | 데스크탑 버블의 상태와 데이터 구성을 맞춘다 |
| 3 | 공개 사이트 세부 섹션 | 발표와 다운로드 안내에 필요하지만 업무 기능보다 뒤다 |

한마디로, 화면 구조는 공개 소개, 회원 업무, 데스크탑 개인 기능을 섞지 않는 것이 핵심이다.
