# 자료보드 요구사항 정리 2026-07-01

## 기준

자료보드는 소개 페이지가 아니라 실제 작업 화면이다. 화면은 개인 자료와 프로젝트룸 자료를 섞지 않는다.
회원 앱 메뉴명은 `자료보드`로 통일한다. 화면 안에서 개인 자료와 현재 프로젝트룸 자료를 전환하되, 두 범위의 입력 방식은 절대 섞지 않는다.

## 개인 자료

- 라우트: `/app/resources`
- 성격: 어느 프로젝트룸을 선택해도 따라오는 내 개인 작업 공간
- 입력 방식: Tauri 앱에서 로컬 폴더를 지정하고 색인한다
- 금지: 업로드 버튼, 드래그앤드롭, 프로젝트룸 멤버 공용 표현
- 보여줄 것: 연결 폴더 상태, 총 파일 수, 최근 스캔, 파일 목록, 파일 미리보기
- 화면 밀도: 통계 카드나 자료 위치 패널로 분산하지 않고, 목록 헤더에 `총 n개`와 연결 상태만 작게 표시한다.
- 전환: 화면 상단의 `개인 / 프로젝트룸` 전환에서 개인이 활성 상태로 보인다
- 빈 상태: `현재 데이터가 없습니다`를 기준으로 조용하게 표시한다
- API: `resourcesApi.listPersonal()`
- 로컬 기능: 폴더 선택과 실제 파일 감지는 Tauri IPC가 맡는다

## 프로젝트룸 자료

- 라우트: `/app/project-rooms/{roomId}/resources`
- 성격: 현재 프로젝트룸 멤버가 함께 보는 공용 자료
- 입력 방식: 파일 선택과 드래그앤드롭 업로드
- 업로드 UI 허용 파일군: PDF, DOC/DOCX, PPT/PPTX, XLS/XLSX, CSV/TSV, TXT, MD/Markdown, HWP/HWPX, PNG/JPG/JPEG/WEBP/GIF/SVG
- 분석 안정 지원은 서버 검증 기준을 따른다. 현재 확실한 분석 대상은 PDF, DOCX, TXT, Markdown이며, 그 밖의 파일은 자료보드 등록과 다운로드/메타데이터 확인을 우선한다.
- 금지: 로컬 폴더 동기화 표현, 개인 자료가 자동으로 공용이 되는 표현
- 보여줄 것: 총 파일 수, 파일 목록, 파일 미리보기, 업로드 진입점
- 화면 밀도: 정리 중/확인 필요 통계 카드는 별도로 만들지 않는다. 정리 상태는 각 파일 행/카드에서 확인한다.
- 업로드 위치: 업로드 카드를 파일 목록 중간에 끼우지 않는다. 상단 액션과 전체 목록 드롭 영역으로 처리한다.
- 전환: 화면 상단의 `개인 / 프로젝트룸` 전환에서 현재 프로젝트룸이 활성 상태로 보인다
- 빈 상태: `현재 데이터가 없습니다`를 기준으로 조용하게 표시한다
- API: `resourcesApi.listRoomResources(roomId)`, `resourcesApi.upload(...)`

## 미리보기

- 파일명만 크게 보이는 박스로 만들지 않는다.
- 실제 파일 본문을 가져오기 전까지는 가짜 본문 줄무늬를 만들지 않는다.
- 파일을 선택하기 전에는 미리보기 패널을 열지 않는다.
- 선택한 파일의 미리보기는 닫기 버튼으로 접을 수 있어야 한다.
- PDF, DOC, PPT, HWP, TXT, MD, 이미지, 표 파일을 구분하되 파일 타입별 색상 아이콘과 원본 파일명, 버전, 크기, 정리 상태, 위치처럼 실제 메타데이터를 먼저 보여준다.
- 공통으로 상태, 최근 수정, 위치, 파일 정보를 보여준다.
- 다운로드는 기존 `resourcesApi.getDownloadUrl(resourceId)`만 사용한다.

## 화면 톤

- `자료보드` 제목은 회원앱의 다른 화면과 같은 Sky Opal 톤의 그라데이션을 쓰되, 랜딩처럼 과장된 히어로 크기로 키우지 않는다.
- 구글 드라이브처럼 범위 전환, 객체/목록 보기, 선택 파일 상세가 한 화면에서 읽혀야 한다.
- 21st.dev file-system 참고 자료는 창 프레임이 아니라 파일/폴더가 놓이는 방식만 참고한다. Bubli 화면 안에서는 브라우저 창처럼 보이는 장식은 만들지 않는다.
- 보기 방식은 두 가지만 둔다. 기본은 객체 카드 보기, 필요할 때 한 줄 목록 보기로 전환한다.
- 객체 보기는 파일 아이콘, 파일명, 형식, 상태를 카드처럼 보여주는 기본 화면이다.
- 목록 보기는 파일을 한 줄씩 빠르게 훑는 스캔용 화면이다.
- 화면 전체의 글자 굵기는 낮춘다. 제목만 힘을 주고, 파일명·상태·메타데이터는 스캔이 쉬운 중간 굵기로 둔다.
- `업로드 상태 대기`처럼 행동을 만들지 않는 상태 문구는 쓰지 않는다.

## 완료 기준

- 개인 자료 화면에 업로드/드래그앤드롭이 없다.
- 프로젝트룸 자료 화면에 로컬 폴더 동기화가 없다.
- 두 화면 모두 검색, 객체/목록 전환, 선택 후 미리보기, 닫기 흐름이 유지된다.
- 자료 위치 패널, 통계 카드, 중간 업로드 카드는 없다.
- `npm run typecheck`, resources 관련 eslint, Playwright 캡처를 통과한다.
