# 회원앱 핵심 플로우 PRD

작성일: 2026-06-30

## 목적

회원 웹 앱은 프리랜서가 프로젝트룸을 고르고, 자료를 정리하고, WBS와 칸반으로 일을 관리하고, 친구와 프로젝트룸 멤버를 초대해 소통하는 화면이다.

지금 화면은 일부 영역에서 설명과 카드가 많아지고, 같은 진입이 반복되며, 실제 사용자가 찾는 핵심 흐름이 약해졌다. 이 문서는 앞으로 화면을 고칠 때 무엇을 남기고 무엇을 줄일지 정하는 기준이다.

참고 링크: [Manyfast feature map](https://manyfast.io/editor/0c33612f-b8dd-4105-be96-3154c4ba28c3?map=featureMap&view=graph)

## 공통 원칙

- 화면은 기능 소개가 아니라 실제 작업 공간이어야 한다.
- 큰 설명문보다 현재 상태, 다음 행동, 바로 갈 수 있는 경로를 우선한다.
- 같은 기능 진입을 여러 카드에서 반복하지 않는다.
- 메뉴로 들어갈 수 있는 상세 기능은 홈에서 자세히 반복하지 않는다.
- 글자 크기, 줄 높이, 카드 밀도는 회원앱 전체에서 맞춘다.
- 서버가 없어도 preview data는 실제 API 응답으로 바꾸기 쉬운 구조여야 한다.
- 삭제된 기능인 링크 초대, 이메일 주소 입력 초대, 비회원 게스트, 매칭, 독립 돈 관리 기능은 되살리지 않는다.

## 자료보드

자료보드는 두 범위를 분명히 보여야 한다.

- 개인 자료: 내 로컬 폴더나 개인 자료함에 연결된 자료
- 프로젝트룸 자료: 지금 선택한 프로젝트룸에 공유된 자료

개인 자료는 업로드 중심 화면이 아니다. Tauri 앱에서 연결한 개인 폴더와 동기화 상태가 중심이다.

프로젝트룸 자료는 업로드와 드래그앤드롭이 가능해야 한다. 단, 항상 “지금 고른 프로젝트룸” 기준임을 보여야 한다.

자료보드에서 필요한 요소:

- 개인 / 프로젝트룸 범위 전환
- 현재 선택된 프로젝트룸 이름
- 개인 자료 동기화 상태
- 프로젝트룸 자료 업로드 영역
- 자료 목록
- 선택한 자료의 분석 상태와 에이전트 정리 결과

자료보드에서 줄일 요소:

- 개인 자료에 프로젝트룸 업로드처럼 보이는 버튼
- 프로젝트룸을 다시 고르라는 카드 반복
- 큰 안내 문단
- 같은 자료 상태를 여러 카드에서 반복 표시

## 프로젝트룸 홈

프로젝트룸 홈은 상세 화면이 아니다. 룸의 현재 상태와 다음 행동만 보여준다.

홈에 남길 요소:

- 프로젝트룸 이름과 기본 정보
- 오늘의 핵심 진행 상태
- 최근 확인 필요 항목
- 멤버 요약
- 자료, 작업판, 소통으로 가는 짧은 진입

홈에서 줄일 요소:

- 자료 목록 상세 반복
- 채팅 미리보기 과다 반복
- 작업판 상세 칸반 반복
- 에이전트 후보 긴 목록
- 메뉴에 이미 있는 상세 기능 카드

## 작업판

작업판은 WBS와 칸반을 모두 포함해야 한다.

필수 구조:

- WBS: 문서에서 나온 작업 구조를 계층 또는 리스트로 보여준다.
- 칸반: 확정된 할 일을 상태별로 보여준다.
- 후보: 에이전트가 만든 WBS/TODO 후보는 승인 전 상태로 분리한다.

작업판에서 피할 것:

- 칸반만 보여서 WBS가 빠진 것처럼 보이는 화면
- 후보와 확정 작업을 같은 목록에 섞기
- 프로젝트룸 홈에서 작업판 상세를 반복하기

## 소통

소통은 프로젝트룸 채팅만이 아니다. 친구 기반 초대 흐름도 보여야 한다.

필수 구조:

- 친구 목록
- 친구 추가
- 수락된 친구를 프로젝트룸에 초대
- 1:1 대화
- 프로젝트룸 채팅
- 프로젝트룸 보이스 진입

초대 기준:

- 프로젝트룸 초대는 수락된 친구만 선택한다.
- 이메일 주소 입력 초대는 만들지 않는다.
- 링크 초대는 만들지 않는다.
- 비회원 게스트 참여는 만들지 않는다.

## 병렬 작업 분리

동시에 작업할 때는 파일을 섞지 않는다.

- 자료보드 트랙: `src/features/resources/**`, `/app/resources`, 프로젝트룸 자료 라우트
- 프로젝트룸/WBS 트랙: `src/app/(workspace)/app/project-rooms/**`, `src/features/project-room/**`, WBS/작업판 관련 파일
- 소통/친구/초대 트랙: `src/app/(workspace)/app/chat/**`, 프로젝트룸 chat 라우트, `src/features/communication/**`
- Tauri 위젯 트랙: `src/app/desktop-widget/**`, `src/features/widget/**`, `src-tauri/**`

## 이번 수정 우선순위

1. 자료보드에서 개인 자료와 현재 프로젝트룸 자료를 명확히 나눈다.
2. 자료보드 글자 크기와 밀도를 회원앱 작업공간 톤에 맞춘다.
3. 프로젝트룸 홈은 다른 세션에서 중복 카드 제거와 다음 행동 중심으로 정리한다.
4. 작업판은 다른 세션에서 WBS와 칸반을 함께 보이게 정리한다.
5. 소통은 다른 세션에서 친구 목록, 친구 추가, 프로젝트룸 초대 흐름을 추가한다.
