# Bubli 기능 연결과 리디자인 체크

작성일: 2026-06-30

이 문서는 비회원 공개 사이트, 회원 전용 웹앱, Tauri 데스크탑 앱과 위젯을 섞지 않기 위한 작업용 체크리스트다. 디자인 수정은 이 표에서 빠진 기능과 잘못 놓인 기능을 확인한 뒤 진행한다.

상태 표기는 `[완료] / [진행] / [예정] / [불일치]`만 쓴다.

## 현재 진행률

| 영역 | 가중치 | 현재 | 판단 |
|---|---:|---:|---|
| 비회원 공개 사이트 | 20% | 65% | 랜딩 톤과 로그인 진입은 잡혔지만 하위 섹션 모션과 다운로드 흐름은 계속 점검 필요 |
| 회원 전용 웹앱 | 45% | 40% | API 호출 구조가 일부 들어갔고 대시보드, 후보, 채팅, 자료, 프로젝트룸 일부가 실제 API 기준으로 정리되기 시작했다. 저장형 대시보드 레이아웃, 프로젝트룸 설정, 룸 작업 세부, 회원앱 전체 톤 정리는 아직 부족 |
| Tauri 앱과 위젯 | 20% | 17% | 위젯 창 상태 IPC와 별도 위젯 창 열기/접기 스캐폴딩을 Rust와 프론트 래퍼에 연결했고 AppShell 없는 `/desktop-widget` surface를 추가했다. 로컬 폴더, SQLite, 트레이는 아직 미연결 |
| API 연결 구조 | 15% | 35% | 래퍼는 넓게 있으나 주요 화면의 실제 데이터 연결 범위는 아직 좁음 |

전체 제품 진행률은 약 35%로 본다. 공개 사이트보다 회원앱과 Tauri 쪽 비중이 크기 때문에 단순 평균보다 낮게 잡는다.

## 공통 원칙

- 공개 사이트는 소개, 다운로드, 로그인 진입만 맡는다.
- 회원 전용 웹앱은 실제 업무공간이다. 소개식 문구보다 바로 쓰는 목록, 필터, 설정, 상세 패널을 우선한다.
- Tauri는 회원 웹앱을 WebView로 띄우고, 데스크탑에서만 가능한 로컬 기능과 위젯을 붙인다.
- 프론트와 Tauri는 항상 API 서버를 호출한다. 에이전트 서버를 직접 호출하지 않는다.
- 디자인보드는 구현 대상이 아니다. 색, 질감, 밀도, 버블감, 모션 참고로만 쓴다.
- 버블은 기능 단위에만 쓴다. 장식용 구슬이나 웹앱 안의 의미 없는 버블은 제거한다.

## 1. 비회원 공개 사이트

| 화면 | 목표 역할 | 현재 상태 | API/연결 | 리디자인 체크 |
|---|---|---|---|---|
| `/` | 제품 이해, 앱 다운로드, 로그인 진입 | [진행] | 업무 데이터 없음 | 랜딩 톤은 맞춰가는 중. 다운로드 CTA는 바로 다운로드 또는 다운로드 섹션 이동이어야 하며 별도 다운로드 페이지 의존을 줄인다 |
| `/features` | 기능 소개 | [예정] | 업무 데이터 없음 | 회원앱 화면을 복사하지 말고 문제와 흐름만 보여준다 |
| `/download` | 앱 다운로드 안내 | [불일치] | 최신 앱 파일 연결 필요 | 별도 페이지가 꼭 필요한지 재검토. 최신 버전만 내려받는 흐름이면 홈 다운로드 섹션과 직접 다운로드가 우선 |
| `/faq` | 최소 질문 정리 | [예정] | 업무 데이터 없음 | 기술 설명보다 사용자가 궁금한 설치, 로그인, 자료 보안 중심 |
| `/login` | Google OAuth 진입 | [진행] | `authApi.getGoogleAuthorizationUrl()` | 랜딩과 같은 폰트, 그라데이션, 배경 버블 톤 유지. 불필요한 인증 설명은 제거 |

비회원 사이트에는 프로젝트룸, 자료보드, 채팅, 보이스, WBS 실제 데이터가 나오면 안 된다.

## 2. 회원 전용 웹앱

| 화면 | 목표 역할 | 현재 상태 | API/연결 | 빠진 것 |
|---|---|---|---|---|
| `/app` | 개인 대시보드 | [진행] | `dashboardApi.getWork()`, Preview seed | 개인 누적, 프로젝트룸 카드, 할 일, 일정, 시간, 자료, 에이전트 후보를 카드 배치 UI로 표시. 저장된 레이아웃, 사용자별 카드 설정 API, 실 타이머 연결은 부족 |
| `/app/project-rooms` | 프로젝트룸 목록과 생성 진입 | [진행] | `projectRoomApi.list()` | 새 프로젝트룸 생성 폼, 계약서 등록 선택, 클라이언트/금액/마감 정보 입력, 나중 수정 |
| `/app/project-rooms/{roomId}` | 프로젝트룸 홈 | [예정] | `projectRoomApi.get()` 필요 | 최근 자료, WBS 요약, 채팅, 멤버, 설정 연결 |
| `/app/project-rooms/{roomId}/resources` | 프로젝트룸 공용 자료 | [진행] | `resourcesApi.listRoomResources(roomId)`, Preview seed | 공용 업로드, 미리보기, 버전, 댓글, 후보 승인. 폴더 동기화 금지 |
| `/app/project-rooms/{roomId}/work` | WBS와 작업판 | [진행] | `wbsApi`, `todoApi` 래퍼 있음 | 후보와 확정 작업 구분, 담당자, 상태 변경, 일정 연결 |
| `/app/project-rooms/{roomId}/chat` | 룸 맥락 채팅 | [불일치] | `chatApi` 래퍼 있음 | 현재 `/app/chat` 중심. 룸 안 채팅 라우트로 정리 필요 |
| `/app/resources` | 개인 자료보드 | [진행] | `resourcesApi`, `projectRoomApi`, Preview seed | Google Drive식 밀도와 파일 미리보기 1차 반영. 후보 승인/수정/보류, 개인 자료 공유 승인 저장은 부족 |
| `/app/schedule` | 일정 | [불일치] | `calendarApi.getEvents()` | 현재 `/app/calendar`. spec 라우트와 이름 정합 필요 |
| `/app/agent` | 에이전트 후보 확인 위치 | [불일치] | `agentApi` 래퍼 있음 | 별도 제안함보다 자료보드, 작업판, 대시보드 안에 후보를 배치하는 방향 검토 |
| `/app/settings` | 계정, 앱, 폴더 설정 | [진행] | `authApi.getMe()`, settings 래퍼 있음 | 실제 설정 저장, Tauri 개인 폴더 연결, 알림 권한 |

### 대시보드 기준

`/app` 첫 화면은 개인 대시보드다. 프로젝트룸 하나의 상세 대시보드가 아니다.

여기에는 개인 누적 정보와 사용자가 고른 프로젝트룸 카드가 함께 놓인다. 예시는 내 TODO, 확인 필요 항목, 오늘 일정, 진행 중 타이머, 프로젝트룸별 작업 시간, 최근 자료, 에이전트 후보 요약이다.

현재 화면은 `src/features/dashboard/data/dashboardPreviewData.ts`의 API 모양 Preview 데이터를 사용해 빈 서버 상태에서도 화면을 볼 수 있게 한다. 이후 백엔드 또는 에이전트 결과가 준비되면 이 파일 또는 같은 타입의 데이터 어댑터를 교체한다.

사이드바는 개인 메뉴와 프로젝트룸 목록을 분리한다. 프로젝트룸 카드 안에는 작업, 자료, 소통 빠른 이동을 둔다. 자료는 `/app/project-rooms/{roomId}/resources`로 연결했고, 소통은 룸 전용 라우트가 생기기 전까지 `/app/chat?roomId={roomId}`로 보낸다.

### 자료보드 기준

| 범위 | 허용 | 금지 |
|---|---|---|
| 개인 자료 | 파일 업로드, 개인 로컬 폴더 동기화, 공유 승인 후 프로젝트룸으로 보내기 | 승인 없이 프로젝트룸 멤버에게 노출 |
| 프로젝트룸 공용 자료 | 업로드, 드래그 앤 드랍, 버전, 댓글, 후보 승인 | 로컬 폴더 동기화 |
| Tauri 개인 로컬 폴더 | 사용자가 앱에서 선택한 폴더만 감지, 켜져 있을 때만 제한 용량 안에서 동기화 | 공용 자료 자동 동기화, 파일 내용 무단 업로드 |

현재 화면은 `src/features/resources/data/resourcePreviewData.ts`의 API 모양 Preview 데이터를 사용한다. 백엔드가 없거나 인증 전이어도 개인 자료, 룸 자료, 로컬 폴더 경계가 보인다. Preview 모드에서는 실제 업로드 버튼을 비활성화한다.

## 3. Tauri 앱과 위젯

| 기능 | 목표 역할 | 현재 상태 | API/IPC | 빠진 것 |
|---|---|---|---|---|
| WebView 메인 창 | 회원 웹앱 `/app` 실행 | [예정] | API 서버 호출 | 실제 Tauri 창 실행, 로그인 토큰 보관 |
| 데스크탑 버블 위젯 | TODO, 에이전트, 소통, 타이머, 메모, 일정, 자료, 알림 | [진행] | Tauri 창 상태 IPC 일부 연결, 버블 타입별 창 라벨, 별도 창 열기/접기 스캐폴딩, AppShell 없는 `/desktop-widget?bubble=...`, 단일 버블 표면, SQLite 표시 캐시 예정 | 버블별 위치/모드 영구 저장, 앱 종료 시 전체 버블 종료 보장, 투명 window effect, Dock Orb 실제 창, 트레이, 단축키 실제 등록, 데이터 연결 |
| 개인 관리 폴더 | 개인 자료 로컬 동기화 | [예정] | Tauri IPC, local sync API | 폴더 선택, 파일 감지, 색인, 업로드 후보 |
| SQLite 캐시 | 채팅, 위젯, 타이머 복구 | [예정] | Tauri IPC 필요 | 테이블 연결, 복구 대기열 |
| 소통 위젯 | 앱 실행 중 채팅/댓글 알림을 위젯 UI로 표시 | [예정] | `chatApi`, `notificationApi`, Tauri 창 필요 | 웹앱 일반 페이지와 분리된 위젯 UI |
| 활동 감지 | 동의 후 앱 이름, 창 제목, 머문 시간 기록 | [예정] | Tauri IPC, activity context API | 권한 UI, 수집 범위 제한, 프로젝트룸 후보 연결 |

Tauri 위젯은 프로젝트룸 화면을 줄여 넣는 곳이 아니다. 개인이 작업 중 바로 확인할 작은 단위만 보여준다.
20v 디자인보드에 있던 Tauri 위젯 기능은 삭제 후보가 아니라 기능 동등성 기준이다.
버블은 각각 독립적인 데스크탑 위젯 단위다. 여러 버블을 한 창 안에 세로 카드로 쌓지 않는다. 단, 모든 버블은 하나의 Tauri 앱 생명주기에 묶이고 앱 종료 시 함께 종료된다.
지금 빠져 보이는 항목은 줄여서 새로 해석하지 않고, 이후 구현에서 채운다.
나머지 회원앱 화면과 세부 디자인은 단계적으로 손보되, Tauri 위젯 범위는 v20보다 작아지면 안 된다.
단, 기획에서 삭제된 받을돈/페이 위젯은 되살리지 않는다.

## 4. API 연결 점검

| 영역 | 래퍼 상태 | 화면 연결 상태 |
|---|---|---|
| 인증 | [완료] `authApi` | 로그인 페이지와 AppShell에서 사용 |
| 대시보드 | [진행] `dashboardApi` | `/app`에서 `GET /api/dashboard/work` 기반 구조와 Preview 데이터를 함께 사용 |
| 프로젝트룸 | [진행] `projectRoomApi` | 목록과 AppShell에서 사용. 생성/수정 UI 부족 |
| 자료 | [진행] `resourcesApi` | 개인/룸 목록, 업로드, 다운로드 URL 사용. `/app/resources`와 `/app/project-rooms/{roomId}/resources`에서 Preview fallback 표시 |
| WBS/TODO | [진행] `wbsApi`, `todoApi` | 라우트 구현 정합 부족 |
| 일정 | [진행] `calendarApi` | `/app/calendar`에서 사용. `/app/schedule`와 불일치 |
| 채팅/소통 | [진행] `chatApi`, `notificationApi` | `/app/chat` 중심. 룸 채팅과 위젯 알림 연결 부족 |
| 위젯/Tauri | [진행] `src/lib/tauri/commands.ts` 창 상태/창 열기 IPC 일부 연결, 나머지는 planned IPC | `/app/desktop/widgets`는 설정/상태 화면. 별도 Tauri 창은 스캐폴딩 단계이며 트레이, SQLite는 미연결 |
| 설정 | [진행] `settingsApi` | 실제 저장 화면 부족 |

API 래퍼와 호출 구조는 있으나, 주요 화면은 아직 일부 데이터와 빈 상태 중심이다. 실데이터 연결은 화면별로 계속 넓혀야 한다.

## 5. 리디자인 작업 순서

| 순서 | 작업 | 상태 | 완료 기준 |
|---:|---|---|---|
| 1 | 회원앱 대시보드를 개인 작업공간으로 재정리 | [진행] | 소개 문구 제거, API 데이터 카드 정리, 확인 필요/타이머/레이아웃 확장 지점 표시 |
| 2 | 자료보드 밀도와 미리보기 강화 | [진행] | 개인/프로젝트룸/로컬 축이 한눈에 보이고, 프로젝트룸 공용은 업로드만 허용 |
| 3 | 프로젝트룸 생성/수정 플로우 | [예정] | 계약서 등록 선택, 기본 정보 입력, 이후 수정 가능 |
| 4 | 프로젝트룸 상세/룸 자료/룸 채팅 라우트 정합 | [예정] | spec 라우트와 현재 코드 라우트 불일치 해소 |
| 5 | Tauri 위젯과 소통 알림 UI | [예정] | 웹앱 일반 페이지가 아니라 데스크탑 위젯 맥락으로 표시 |

각 작업은 시작 전 이 문서에서 빠진 항목을 확인하고, 작업 후 상태를 갱신한다.
