## 제품 기준

소통 화면은 소개 화면이 아니다. 사용자가 실제로 프로젝트룸 안에서 대화하고, 친구와 1:1을 열고, 보이스 참여 상태를 확인하는 작업 공간이다.

프로젝트룸 대화는 프로젝트룸에 귀속된다. 사용자가 특정 프로젝트룸을 선택한 상태라면 대화 화면도 그 룸 맥락을 유지한다. 같은 화면에서 다른 프로젝트룸 대화를 임의로 섞어 보여주지 않는다.

친구는 1:1 대화와 프로젝트룸 초대의 전제다. 이메일 초대, 링크 초대, 비회원 게스트 초대는 만들지 않는다. 친구 추가는 Bubli ID 검색, 친구 요청 보내기, 받은 요청 수락 또는 거절 흐름으로 처리한다.

보이스는 프로젝트룸 보이스를 우선 구현한다. 1:1 보이스는 같은 구조를 재사용할 수 있는 확장 후보로 두되, 현재 화면에서는 준비 상태를 분명히 표시한다.

## 화면에 반드시 보여야 하는 것

| 영역 | 필수 요소 | 기준 |
|---|---|---|
| 대화 목록 | 프로젝트룸 대화, 1:1 대화 전환 | 프로젝트룸 진입 상태에서는 현재 룸 대화를 우선 표시 |
| 메시지 | 보낸 사람, 시각, 본문, 메시지 보조 액션 | 메시지는 서버 `roomSequence` 순서 기준 |
| 입력창 | 파일 첨부, 보이스 상태 열기, 본문 입력, 전송 | 내가 입력 중일 때 상대 입력 중 표시를 띄우지 않음 |
| 친구 패널 | 내 Bubli ID 복사, Bubli ID 검색, 친구 요청 보내기 | Discord의 태그 복사 흐름을 Bubli ID 기준으로 단순화 |
| 요청 패널 | 받은 요청 수락/거절, 보낸 요청 상태 | 수락된 친구만 1:1과 초대 대상 |
| 보이스 패널 | 참여자 아바타, 참여 상태, 말하는 사람 강조 | `JOINED`, `LEFT`, `DISCONNECTED` 상태를 화면에서 구분 |

## 참고 UI 해석

사용자가 제공한 참고 컴포넌트는 그대로 복사하지 않고 아래 역할만 가져온다.

| 참고 | 가져올 것 | Bubli 적용 |
|---|---|---|
| audio-chat | 접힌 참여자 스택, 펼친 참여자 목록, 말하는 사람 강조 | 보이스 패널에서 참여자 상태와 말하는 사람을 표시 |
| chat-template | 왼쪽 목록, 가운데 대화, 하단 입력의 기본 구조 | 소통 화면을 목록, 메시지, 사람 패널 3영역으로 정리 |
| messaging-conversation | 메시지 카드와 hover 액션 | 메시지 복사, 더보기 같은 보조 액션을 낮은 강조로 표시 |
| ai-chat-input | 입력창이 focus될 때 도구가 자연스럽게 확장 | 첨부, 보이스, 짧은 응답 선택을 입력 흐름 안에 배치 |

## API 연결 기준

| 기능 | API | 현재 화면 연결 |
|---|---|---|
| 친구 목록 | `GET /api/friends` | 친구 패널 |
| 친구 검색 | `GET /api/friends/search?bubliId={id}` | 친구 추가 검색 |
| 친구 요청 보내기 | `POST /api/friend-requests` | 검색 결과 요청 버튼 |
| 요청 목록 | `GET /api/friend-requests` | 친구 요청 패널 |
| 요청 수락/거절 | `PATCH /api/friend-requests/{id}/accept`, `PATCH /reject` | 받은 요청 버튼 |
| 대화방 목록 | `GET /api/chat/rooms` | 대화 목록 |
| 1:1 방 열기 | `POST /api/chat/direct-rooms` | 친구 행 클릭 |
| 메시지 목록 | `GET /api/chat/rooms/{id}/messages` | 메시지 영역 |
| 메시지 전송 | `POST /api/chat/rooms/{id}/messages` | 입력창 전송 |
| 프로젝트룸 보이스 시작 | `POST /api/voice/rooms` | 보이스 시작 |

## 구현 체크리스트

- [완료] 내 Bubli ID 복사 위치를 소통 화면에 둔다.
- [완료] Bubli ID 검색과 친구 요청 보내기를 같은 패널에 둔다.
- [완료] 받은 친구 요청 수락/거절을 같은 화면에서 처리한다.
- [완료] 프로젝트룸 URL에서 들어온 대화는 현재 프로젝트룸 대화로 고정한다.
- [완료] 보이스 참여자 스택과 펼친 참여자 목록을 둔다.
- [진행] 1:1 보이스는 API 확정 후 같은 보이스 UI로 연결한다.
- [예정] WebSocket 이벤트를 붙여 새 메시지, 읽음, 보이스 참여 상태를 자동 갱신한다.
- [예정] 프로젝트룸 초대 화면에서 수락된 친구만 선택할 수 있게 연결한다.
