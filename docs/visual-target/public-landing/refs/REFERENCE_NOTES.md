# 공개 랜딩 레퍼런스 노트 (2026-06-29)

브라우저(Chrome)로 직접 접속해 본 레퍼런스 분석. **브라우저 스크린샷은 디스크 저장이 안 돼서** 원본 컷은 못 남기고 분석만 고정한다. 색은 전부 Bubli Sky Opal로 치환(청록/민트/aqua/teal 금지).

## 사이트별 추출

### Voiceflow (voiceflow.com)
- 큰 **editorial 세리프 헤드라인** 2줄 중앙 정렬, 회색에 가까운 밝은 배경, 넉넉한 여백.
- 블루 **pill CTA** 2개(채움 "Book a demo" + 외곽선 "Sign up").
- 헤드라인 아래 **로고 스트립**(social proof), 그 아래 대형 풀블리드 비주얼.
- 적용: Bubli 히어로도 세리프 헤드라인 "받은 자료를, 오늘의 할 일로." + Sky/Bubble Blue pill CTA + 로고/지표 스트립.

### Kastle (kastle.ai)
- 중앙 세리프 헤드라인 + 한 줄 서브 + 외곽선 pill CTA 2개.
- 핵심: **제품 대시보드 목업을 히어로 하단에 크게** (Analytics Dashboard, 보라 area chart).
- 적용: 히어로 아래 **Bubli 대시보드/위젯 목업**(오늘 요약·프로젝트 시간 링·에이전트 제안)을 Sky Opal로. 보라 차트 → sky→lilac.

### ToDesktop (todesktop.com)
- **다크 네이비 히어로** + 굵은 산세리프 헤드라인 + 한 줄 서브 + 블루 채움 CTA.
- **3열 기능 아이콘**(Upload / Secure / Distribute) + 글래스 프레임 터미널/데스크탑 목업.
- 웹앱→데스크탑 = Bubli Tauri와 직접 동일 맥락.
- 적용: **다크 변형 히어로(Night Bubble)** + 3열 기능 + Bubli 데스크탑 위젯 목업을 글래스 프레임으로.

### Spline Community — blobs (community.spline.design, query=blobs)
- 3D blob/오브 씬 갤러리(다크 UI). Bubli 버블 브랜드 모션의 소스.
- 적용: 히어로의 **절제된 3D 버블/blob 모션 액센트**(과하지 않게). 실제 사용 시 씬 선택 → 임베드 또는 영상 추출. 일부는 로그인/저작자 권리 확인 필요.

### Drift (기존 보유)
- Drifty식 좌측 내비 + "Good morning" 톤 정보 구조. 시간 흐름은 Bubli식 **프로젝트 귀속**으로 변환.

## Bubli 랜딩 합성 방향

1. 히어로: 세리프 헤드라인 + Sky/Bubble Blue pill CTA + 옅은 Sky Opal 그라데이션/홀로그래픽 배경. (라이트 기본)
2. 히어로 하단: **제품 목업**(대시보드 또는 데스크탑 위젯) 글래스 프레임.
3. 다크 변형: Night Bubble 히어로(todesktop처럼 깊은 네이비 + 낮은 채도 glow + 데스크탑 위젯 목업).
4. 모션 액센트: 절제된 3D 버블/오팔 blob(Spline 무드), 배경을 잡아먹지 않게.
5. 섹션 리듬: 로고 스트립 → 기능 3열 → 흐름(자료→후보→승인→오늘 할 일) → 다운로드(데스크탑 앱) CTA.

색은 Sky Opal만. 큰 장식 버블은 **랜딩에서는 허용**(회원 웹앱과 다름) 하되 톤은 맑게.

## 제약(중요)
- 브라우저 스크린샷은 이 세션에서 디스크 저장 불가 → 원본 레퍼런스 컷은 사용자가 직접 캡처/저장해야 함.
- 현재 연결 브라우저가 **Browser 2(Windows·원격)** 라, 여기서 생성·다운로드하는 이미지/영상은 Windows에 떨어진다(이 레포는 맥). 생성물을 레포로 넣으려면 맥 브라우저(Browser 1) 사용 또는 수동 이동 필요.
