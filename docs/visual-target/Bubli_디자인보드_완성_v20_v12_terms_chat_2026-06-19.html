Bubli 디자인보드 v20 기준안

v19를 복제한 뒤 v12 기획과 맞는 화면만 남긴 정리본이다. 중복 자료보드, WBS/TODO, 위젯 실험은 화면 기준에서 제외하고 아래 기존 보드에 필요한 인터랙션만 합쳤다. 새로 추가한 데스크탑 프리뷰는 Tauri 앱에서 위젯이 실제 바탕화면 위에 뜰 때의 기준 장면이다.

v12 적용 판정 — 최종 기획과 참고 이미지를 어떻게 합칠지

예쁜 이미지를 그대로 베끼지 않는다. v12 기획과 맞는 질감과 정보 구조만 가져오고, 대시보드와 위젯처럼 해석이 틀어진 부분은 화면 안에서도 보류 또는 금지로 표시한다.

공개 사이트히어로와 3D 버블은 유지한다. 소개와 다운로드 역할만 맡는다.
자료보드상단 실험 화면은 폐기했다. 아래 기존 보드 안에서 개인, 프로젝트룸, 권한 범위를 함께 본다.
WBS/TODO별도 중복 실험 화면은 숨겼다. 아래 승인 흐름과 작업판 구조로 합친다.
위젯v13 글라스 감각을 기준으로 한다. 웹 설정 페이지처럼 보이는 화면은 쓰지 않는다.
대시보드최종 카드 배치는 보류한다. 지금은 데이터 계약과 연결 흐름만 보여준다.

바로 가져갈 것

브랜드 첫인상과 핵심 화면 구조에 바로 반영한다. 화면의 뼈대가 v12 기획과 맞는 요소다.

큰 투명 Bubli 방울 마크와 얇은 유리 림 자료 목록, 에이전트 정리, 확인 항목, 관련 문서, 댓글의 오른쪽 판단 패널 WBS 트리, 칸반 작업판, 에이전트 제안 후보의 3열 작업 구조

고쳐서 가져갈 것

시각 질감은 좋지만 제품 해석이 일부 어긋난 요소다. 문구와 역할을 고쳐서 쓴다.

대시보드는 카드 밀도와 가독성만 참고하고 최종 배치는 보류한다. 위젯 화면은 웹 설정 페이지가 아니라 Tauri 바탕화면 프리뷰로 바꾼다. 금액 정보는 계약 확인 보조 정보로만 둔다.

가져오면 안 되는 것

기획을 흐리거나 4주 구현 범위를 키우는 방향이다. 화면 제작 때 다시 살리지 않는다.

대시보드를 계약 금액 중심 화면으로 고정하는 구성 위젯을 별도 웹 페이지처럼 크게 설명하는 구성 에이전트가 사용자 승인 없이 업무를 확정하는 듯한 문구
공개 사이트 히어로

유리 카드중앙 프리뷰설정 패널

데스크탑 1200px 이상

  • 디자인보드 폭은 1180px 기준으로 둔다.
  • 자료보드는 240px 탐색, 1fr 목록, 360px 상세 패널로 본다.
  • WBS 작업판은 240px 트리, 1fr 칸반, 300px 제안 패널을 기준으로 한다.

태블릿 760px 이상

  • 사이드바는 아이콘 레일로 줄이고 주요 화면은 1열 또는 2열로 바꾼다.
  • 자료보드 상세 패널은 목록 아래로 접거나 슬라이드 패널로 연다.
  • WBS 트리는 상단 탭 또는 접힘 패널로 바꾼다.

모바일 390px 기준

  • 공개 사이트는 한 화면에 히어로, 버튼, 다음 섹션 힌트만 둔다.
  • 회원 앱은 목록 먼저, 상세는 별도 화면 또는 하단 패널로 연다.
  • 위젯 버블은 실제 모바일 제품이 아니라 작은 창 규격 검토용으로만 본다.
랜딩 방울3D 느낌을 가장 크게 쓴다. 6초 이상 느린 부유, 낮은 대비, 텍스트와 겹치지 않는 위치가 기준이다.
에이전트 상태분석 중에는 작은 pulse만 쓴다. 업무 판단을 방해하는 튐, 회전, 강한 색 변화는 쓰지 않는다.
위젯 버블호버 때 림과 그림자만 살아난다. 항상 움직이는 위젯은 작업 중 거슬릴 수 있으므로 기본은 정적이다.
접근성prefers-reduced-motion과 앱 설정에서 모션 끄기를 둔다. 끄면 모든 부유와 pulse가 멈춘다.
컴포넌트 분리버블 이미지는 에셋, 움직임은 transform, 상태 빛은 별도 레이어로 둔다. 나중에 BubbleMark, FloatingBubble, WidgetGlassCard로 나눌 수 있게 만든다.

5

제품 구조 — 자료보드 범위 필터

자료보드 범위 필터는 따로 있는 화면이 아니다. v12 기준으로 자료보드 안에서 개인 자료, 프로젝트룸 자료, 확인할 항목을 전환해 보는 필터이자 권한 표시다. 개인 관리 폴더와 개인 자료함 동기화는 프로젝트룸 자동 공유가 아니라 사용자 개인 영역으로 분리한다.

비회원 페이지서비스 소개와 앱 다운로드. 로그인 전에는 제품의 핵심 흐름만 보여준다.
로그인 후 웹대시보드, 자료보드, 프로젝트룸, 설정을 웹에서 쓴다.
Tauri 데스크탑 앱같은 웹 화면을 앱으로 감싸고, 위젯, 개인 관리 폴더, 로컬 SQLite를 붙인다.
데스크탑 위젯작업 중에는 TODO, 에이전트, 소통, 타이머, 메모, 일정/WBS, 자료 제안, 알림을 버블로 본다.

대시보드

프로젝트룸마다 따로 생기는 화면이 아니다. 여러 프로젝트룸에서 내가 맡은 TODO, 일정, 확인 필요 항목, 작업 시간을 모아 보는 사용자 기준 화면이다. 카드 구성은 아직 회의 보류다.

폰트와 테마 계획

폰트가 작다는 피드백을 반영해 본문 기준을 15px로 올리고, 사용자가 글자를 확대하거나 축소할 수 있는 설정을 둔다. 위젯 최소 본문은 12.5px 아래로 내려가지 않게 하고, 다크모드는 검정에 가까운 표면과 흰 글자, 하늘색 강조만 남기는 블랙화이트 다크 테마로 잡는다.

사용 폰트 기본은 Pretendard Variable 하나로 통일한다. 한국어 화면, 숫자, 버튼, 위젯을 같은 글꼴로 맞춰서 제품이 흩어져 보이지 않게 한다. Pretendard Variable
공개 사이트 첫인상 화면은 33-72px까지 크게 쓴다. 굵기는 840-880을 쓰되 자간은 0으로 두고, 버블 이미지가 주인공이 되게 문장 수를 줄인다.
회원 앱 업무 화면은 15px 본문, 21px 섹션 제목, 12-13px 라벨을 기준으로 한다. 정보 밀도는 높이되 줄간격을 넉넉하게 둔다.

고스트 모드 글자 기준

고스트는 위젯 배경을 거의 지우고 글자만 남기는 단계다. 밝은 벽지는 어두운 글자와 흰 halo, 어두운 벽지는 흰 글자와 어두운 halo를 쓴다. 배경이 달라도 업무 내용이 먼저 읽히는지 직접 확인한다.

규칙 7가지

  • 밝은 벽지는 어두운 글자에 흰 halo를 준다. 흰 배경에서 검은 외곽선이 튀는 문제를 막는다.
  • 어두운 벽지는 흰 글자와 어두운 그림자를 쓴다. 바탕화면 색이 복잡해도 글자 경계가 남는다.
  • 글자 굵기는 한 단계 올린다(640 → 760). 가는 글자는 외곽선에 먹힌다.
  • 최소 글자 크기는 13px. 캡션(11.5px)과 시간(10px)은 고스트에서 한 단계 승급한다.
  • 자간 +0.2px, 숫자는 tabular-nums로 흔들림을 막는다.
  • 위계는 투명도가 아니라 그림자 강도로 나눈다. 보조 글자도 흰색을 유지하고 그림자만 약하게.
  • 버튼, 입력칸, 드롭존, 체크박스는 숨긴다. 메모 스티커는 색이 곧 의미라 옅은 노랑을 유지한다.
  • 클릭은 위젯을 통과한다. 머리줄만 살아 있고, 호버하면 컨트롤 캡슐이 떠오른다.
