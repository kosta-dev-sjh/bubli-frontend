# Bubli 공개(비회원) 사이트 요구사항 (2026-06-29)

이 문서는 공개 사이트 구현의 기준이다. 작업할 때 계속 참고하고, 바뀌면 여기부터 고친다.

## 0. 범위 / 정체성

- 공개(비회원) 사이트 = **소개 + 다운로드 + 구글 로그인 진입**. 업무 데이터·회원 기능 없음.
- 별도 이메일 회원가입(`/signup`) 없음. "회원가입/로그인" = **구글 OAuth 진입**(실제 OAuth 연결은 아직 X, 화면만).
- **라이트 모드 전용.** 공개 페이지에서 다크 모드는 뺀다(회원앱만 다크 유지).

## 1. 구조 — 단일 페이지 스크롤 랜딩

레퍼런스: Voiceflow(와이드 레이아웃 + 상단 메뉴바), getanchor.co / todesktop.com(스크롤 내리면 앱 설명이 순차 등장).

- **상단 고정 메뉴바(top bar)**: 와이드 구성. 스크롤해도 상단 고정(sticky), 스크롤 시 배경 글래스/축소 등 CSS 반응.
- **메뉴 = 앵커 링크.** 클릭 시 별도 페이지로 가지 않고 **같은 페이지의 해당 섹션으로 스크롤 이동**(`#기능`, `#다운로드`, `#faq` 등). 부드러운 스크롤(smooth), `prefers-reduced-motion` 존중.
- **본문 = 스크롤 리빌 섹션**(아래로 내리면 등장):
  1. Hero — 핵심 문장 + 3D/글래스 버블 + CTA(앱 다운로드 / 구글 로그인 진입)
  2. 문제 → 해결 (왜 필요한가)
  3. 핵심 기능 / 화면 미리보기 (프로젝트룸·자료보드·대시보드·버블)
  4. 데스크탑 앱 소개 (Tauri 위젯)
  5. 다운로드 (페이지 내)
  6. FAQ
  7. 푸터
- **다운로드**: 페이지 내 다운로드 섹션에서 직접. **최신 버전만** 다운로드 가능(mac / windows). 과거 버전 목록 없음.

## 2. 라우트 / 네비

- 메인은 `/` 단일 페이지. 메뉴는 `/#기능`, `/#다운로드`, `/#faq` 같은 앵커.
- 기존 `/features` `/download` `/faq` 라우트는 유지(직접 진입/SEO 대비)하되, **상단 메뉴는 `/`의 섹션 앵커**를 가리킨다. (라우트 구조는 깨지 않음.)
- 구글 로그인 진입은 `/login`(또는 상단바 버튼) — 실제 OAuth 연결 금지 유지.

## 3. 디자인 / 인터랙션

- 디자인 시스템 준수: LINE Seed Sans KR 폰트, Sky Opal 글래스, 넉넉한 여백(airy), 작고 맑은 shadow.
- 색: 청록/민트/aqua/teal 금지. 색은 면이 아니라 rim/ring/glow.
- 히어로 버블: 라이트 배경에서 보이도록 **컬러 글래스 디스크 + R3F(Three.js) 광택·iridescence·float**. (회원앱에는 큰 장식 버블 금지, 랜딩 히어로만 허용.)
- 스크롤 리빌: 섹션 진입 시 fade/blur-up/translate. `prefers-reduced-motion`이면 즉시 표시.
- 상단바: 스크롤 위치에 따라 글래스 강도/그림자 변화.

## 4. 레퍼런스

- 레이아웃·탑바: https://www.voiceflow.com/
- 스크롤 설명형: https://getanchor.co/ , https://www.todesktop.com/
- 인터랙션 컴포넌트: Magic UI, Motion Primitives (docs/UI_Reference_Library.md 참고)

## 5. 완료 기준

- `/`에서 상단 메뉴 클릭 → 해당 섹션으로 스무스 스크롤.
- 아래로 스크롤 → 섹션이 순차로 등장.
- 다운로드 섹션에서 mac/win 최신 버전 버튼.
- 라이트 전용(다크로 깨지지 않음).
- `route-home-light.png` 캡처 확인, typecheck/visual:qa 통과, 금지색 0.

## 6. 구현 메모 / 진행

- [완료] 라이트 전용 강제(ForceLightTheme), LINE Seed 폰트, 히어로 컬러 글래스 디스크 + R3F 버블(라이트에서 보임), 스크롤 리빌 컴포넌트(Reveal).
- [예정] 상단 고정 메뉴바(앵커 nav + 스크롤 반응), 단일 페이지 섹션 구성(문제/기능/앱/다운로드/FAQ), 다운로드 섹션(최신만), 스무스 앵커 스크롤.
- 주의: 헤드리스(visual:qa) 캡처는 WebGL 버블이 흐리게 잡힌다 → 라이브(localhost:3000) 또는 headful 캡처로 확인.
