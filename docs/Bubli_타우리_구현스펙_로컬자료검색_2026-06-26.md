## 0. 전제 — 경계는 이미 우리 기획에 못박혀 있다

프론트 레포 `src/lib/tauri/commands.ts`에 IPC 계약이 이미 타입으로 정의돼 있다. 즉 "타우리가 무엇을 하는가"는 우리 설계에서 이미 결정된 상태이고, **Rust 구현(`src-tauri/src/*`가 전부 빈 `.gitkeep`)만 비어 있다.** 이 스펙은 그 빈칸을 채우는 일이다.

이미 선언된 계획 커맨드(`PLANNED_TAURI_COMMANDS`): `select_managed_folder`, `scan_managed_folder`, `watch_managed_folder`, `search_local_files`, `read_activity_context`, `flush_sync_outbox`, `recover_timer_state`, `backup_local_sqlite`, `restore_local_sqlite_backup`, `check_local_sqlite_integrity`, `record_widget_usage_event`, `rollup_widget_usage`, `sync_widget_usage_summary`, `sync_room_messages`.

특히 `search_local_files`의 결과 타입에 이미 `matchedText`가 있다. 이건 파일명만이 아니라 **본문 내용 검색**을 처음부터 의도했다는 뜻이다. Docufinder에서 가져올 부분이 정확히 여기다.

### 0.1 타우리 vs 백엔드 경계 (확정)

| 일 | 담당 | 근거 |
|---|---|---|
| 폴더 선택·스캔·실시간 감시 | **타우리** | 기기 파일 접근은 네이티브만 가능. `select/scan/watch_managed_folder` |
| 로컬 파일 메타·본문 색인(SQLite FTS5) | **타우리** | 개인 원문 비업로드 원칙. `local_files`, FTS5 |
| 로컬 파일명·본문 즉시 검색 | **타우리** | 오프라인 즉시 응답. `search_local_files`(matchedText) |
| 활동 인식(앱·창·시간) | **타우리** | 네이티브 OS API. `read_activity_context` |
| 서버 반영 대기열 전송 | **타우리→백엔드** | `flush_sync_outbox` → localsync API |
| 파일 저장(S3), 자료 메타 원본 | **백엔드** | `POST /api/resources`(multipart) |
| 텍스트 추출→chunk→embedding(pgvector) | **백엔드** | 무거운 AI는 서버. RAG 명세 1~3단계 |
| 의미검색·관련자료·요약·후보 | **백엔드** | `/summary`, `/related`, `/api/ai/*`, `agent_suggestions` |

한마디로, **타우리는 "내 PC 안에서 빠르게 찾고 감시하고 추출"하고, 백엔드는 "올린 자료를 임베딩·RAG·후보"한다.** 둘이 자료보드 검색창에서 합쳐진다.

## 1. 의존 크레이트 (Cargo.toml에 추가)

현재 `Cargo.toml`은 `tauri`, `tauri-plugin-opener`, `serde`만 있다. 아래를 추가한다. (버전은 착수 시 `cargo add`로 최신 확인)

```toml
[dependencies]
# 이미 있음: serde, serde_json, tauri, tauri-plugin-opener
tauri-plugin-dialog = "2"          # 폴더 선택 다이얼로그
tauri-plugin-fs = "2"              # 파일 접근(권한 capability와 함께)
tauri-plugin-notification = "2"    # 알림
rusqlite = { version = "0.32", features = ["bundled"] }  # 번들 SQLite = FTS5 포함
notify = "6"                       # 파일시스템 감시
notify-debouncer-full = "0.3"      # 감시 이벤트 디바운스(중복 방지)
walkdir = "2"                      # 폴더 재귀 스캔
sha2 = "0.10"                      # 파일 checksum(변경 감지)
encoding_rs = "0.8"                # EUC-KR/CP949 텍스트 디코딩
chardetng = "0.1"                  # 인코딩 자동 감지
uuid = { version = "1", features = ["v4"] }
chrono = "0.4"
# 텍스트 추출
calamine = "0.26"                  # xlsx/xls (시트·행 위치)
zip = "2"                          # docx/pptx 언집
quick-xml = "0.36"                 # docx/pptx 본문 XML 파싱
pdf-extract = "0.7"                # pdf 텍스트 레이어 추출
# 활동 인식(Drifty)
# x-win = "1"                      # 활성 앱·창·URL (macOS는 접근성 권한 필요)
reqwest = { version = "0.12", features = ["json", "multipart"] }  # outbox→백엔드 전송
```

선택 결정 두 가지:
- **SQLite 접근은 Rust `rusqlite`로 단일 소유.** 프론트 `@tauri-apps/plugin-sql`도 깔려 있지만, FTS5 색인·검색은 Rust가 직접 소유하고 프론트는 IPC로만 부른다. DB 소유자를 한쪽으로 모아야 잠금·무결성 사고가 안 난다.
- **한국어 FTS는 FTS5 `trigram` 토크나이저로 시작.** Lindera 형태소 분석은 품질은 좋지만 사전 번들이 무겁다. 4주엔 trigram(부분일치, CJK에 충분히 동작)으로 가고, Lindera는 스트레치로 둔다.

## 2. 로컬 SQLite 스키마 (DDL)

`src-tauri/src/sqlite/schema.sql`. 우리 09C 설계(`managed_folders`, `local_files`, `local_file_events`, `local_sync_outbox`)를 그대로 구현하고 FTS5 본문 색인을 더한다.

```sql
-- 등록한 폴더
CREATE TABLE IF NOT EXISTS managed_folders (
  local_folder_id TEXT PRIMARY KEY,
  name         TEXT NOT NULL,
  path         TEXT NOT NULL UNIQUE,
  sync_enabled INTEGER NOT NULL DEFAULT 0,           -- 0=로컬만, 1=서버 동기화
  status       TEXT NOT NULL DEFAULT 'ACTIVE',       -- ACTIVE/PAUSED/REMOVED
  created_at   TEXT NOT NULL
);

-- 파일 메타
CREATE TABLE IF NOT EXISTS local_files (
  local_file_id   TEXT PRIMARY KEY,
  local_folder_id TEXT NOT NULL REFERENCES managed_folders(local_folder_id),
  path        TEXT NOT NULL,
  name        TEXT NOT NULL,
  ext         TEXT,
  size_bytes  INTEGER,
  checksum    TEXT,                                  -- sha256, 변경 감지
  modified_at TEXT,
  indexed_at  TEXT,
  extract_status TEXT NOT NULL DEFAULT 'PENDING',    -- PENDING/DONE/UNSUPPORTED/FAILED
  sync_status    TEXT NOT NULL DEFAULT 'LOCAL_ONLY', -- LOCAL_ONLY/SYNC_PENDING/SYNCED
  status      TEXT NOT NULL DEFAULT 'ACTIVE',        -- ACTIVE/DELETED
  UNIQUE(local_folder_id, path)
);
CREATE INDEX IF NOT EXISTS idx_local_files_folder ON local_files(local_folder_id);

-- 본문 전문 색인 (rowid = local_files.rowid 매핑)
CREATE VIRTUAL TABLE IF NOT EXISTS local_file_fts USING fts5 (
  name,
  content,
  path UNINDEXED,
  tokenize = 'trigram'
);

-- 변경 이벤트 (서버 반영 대기)
CREATE TABLE IF NOT EXISTS local_file_events (
  event_id   TEXT PRIMARY KEY,
  local_file_id   TEXT,
  local_folder_id TEXT NOT NULL,
  event_type TEXT NOT NULL,                          -- CREATED/MODIFIED/DELETED/MOVED
  path       TEXT NOT NULL,
  old_path   TEXT,
  occurred_at TEXT NOT NULL
);

-- 서버 반영 대기열 (idempotency)
CREATE TABLE IF NOT EXISTS local_sync_outbox (
  outbox_id       TEXT PRIMARY KEY,
  idempotency_key TEXT NOT NULL UNIQUE,
  endpoint    TEXT NOT NULL,
  payload_json TEXT NOT NULL,
  status      TEXT NOT NULL DEFAULT 'PENDING',       -- PENDING/SENT/FAILED
  retry_count INTEGER NOT NULL DEFAULT 0,
  created_at  TEXT NOT NULL
);

-- 활동 인식 로컬 버퍼 (서버 activity_logs로 동기화)
CREATE TABLE IF NOT EXISTS local_activity_buffer (
  id TEXT PRIMARY KEY,
  app_name TEXT, window_title TEXT, room_id TEXT,
  started_at TEXT NOT NULL, ended_at TEXT, duration_seconds INTEGER,
  synced INTEGER NOT NULL DEFAULT 0
);
```

FTS5는 `trigram`이라 부분 문자열·한국어가 걸린다. 본문 검색 결과 스니펫은 `snippet(local_file_fts, ...)`로 뽑아 `matchedText`에 담는다.

## 3. IPC 커맨드 명세 (Rust 시그니처 ↔ 기존 TS 계약)

각 커맨드는 `commands.ts`의 타입과 1:1로 맞춘다. Rust 반환 구조체는 `#[serde(rename_all = "camelCase")]`로 TS의 camelCase에 맞춘다.

| TS 커맨드 | Rust 시그니처 | 모듈 | 하는 일 |
|---|---|---|---|
| `select_managed_folder` | `fn select_managed_folder(app) -> Result<ManagedFolderSelection>` | commands/folder | 다이얼로그로 폴더 선택 → `managed_folders` insert |
| `scan_managed_folder` | `fn scan_managed_folder(id: String) -> Result<ManagedFolderScanResult>` | folder_watch | 재귀 스캔, checksum diff, `local_files` upsert, 추출 큐 적재 |
| `watch_managed_folder` | `fn watch_managed_folder(id: String) -> Result<ManagedFolderWatchResult>` | folder_watch | notify 감시 시작, 이벤트→`local_file_events`+재색인 |
| `search_local_files` | `fn search_local_files(query, limit) -> Result<LocalFileSearchResult>` | search | FTS5(name+content) 질의, snippet→matchedText |
| `read_activity_context` | `fn read_activity_context() -> Result<ActivityContextResult>` | activity | 현재 활성 앱·창·시간 |
| `flush_sync_outbox` | `fn flush_sync_outbox(app) -> Result<SyncOutboxFlushResult>` | sync | outbox PENDING을 백엔드 localsync로 전송 |
| `check_local_sqlite_integrity` | `fn ... -> Result<SqliteIntegrityResult>` | sqlite | `PRAGMA integrity_check` |

신규로 추가할 커맨드(‎`commands.ts`에도 타입 추가 필요):

| 신규 커맨드 | 시그니처 | 용도 |
|---|---|---|
| `get_index_progress` | `fn(id) -> IndexProgress { total, indexed, pending }` | 색인 진행률 UI |
| `reindex_file` | `fn(localFileId) -> ()` | 단일 파일 재추출 |
| `open_local_file` | `fn(localFileId) -> ()` | 결과 더블클릭 시 OS로 열기(opener) |
| `set_folder_sync` | `fn(id, enabled) -> ()` | 폴더 서버 동기화 on/off |

추출(`extract_*`)은 IPC가 아니라 스캔/감시 뒤 백그라운드 태스크로 돈다(아래 5.4).

## 5. 핵심 흐름 (시퀀스 + Rust 스켈레톤)

### 5.1 폴더 등록 → 스캔 → 본문 색인

```
[프론트] selectManagedFolder()
  -> dialog로 폴더 선택
  -> managed_folders insert
[프론트] scanManagedFolder(id)
  -> walkdir로 폴더 순회
  -> 파일별 checksum 계산, local_files와 비교
       신규/변경: upsert + extract_status=PENDING
       삭제: status=DELETED + FTS 삭제
  -> 변경 건수 반환 (즉시 파일명 검색 가능)
[백그라운드] PENDING 파일을 추출 태스크가 처리
  -> extract::run(path, ext) -> 본문 텍스트
  -> index::fts upsert(name, content)
  -> extract_status=DONE, 진행률 이벤트 emit
```

스캐너 스켈레톤:

```rust
pub fn scan(db: &Db, folder: &ManagedFolder) -> Result<ScanResult> {
    let mut changed = 0;
    for entry in WalkDir::new(&folder.path).into_iter().filter_map(|e| e.ok()) {
        if !entry.file_type().is_file() { continue; }
        let path = entry.path();
        let meta = entry.metadata()?;
        let checksum = sha256_head(path, 1 << 20)?;   // 앞 1MB만 해시(속도/정확 균형)
        match db.find_file(&folder.local_folder_id, path)? {
            Some(f) if f.checksum == checksum => {}    // 변화 없음
            _ => {
                db.upsert_local_file(&folder.local_folder_id, path, &meta, &checksum)?;
                db.enqueue_event(&folder.local_folder_id, path, "MODIFIED")?;
                changed += 1;
            }
        }
    }
    Ok(ScanResult { changed_count: changed, scanned_at: now() })
}
```

추출 라우팅:

```rust
pub fn extract(path: &Path, ext: &str) -> ExtractOutcome {
    match ext.to_ascii_lowercase().as_str() {

    }
}
```

### 5.3 로컬 검색 (FTS5 + snippet)

```rust
#[tauri::command]
pub fn search_local_files(db: State<Db>, query: String, limit: Option<u32>)
  -> Result<LocalFileSearchResult>
{
    let limit = limit.unwrap_or(30);
    // name 또는 content 매칭. snippet으로 matchedText 생성.
    let sql = "SELECT f.local_file_id, f.name, f.path, f.modified_at,
                      snippet(local_file_fts, 1, '[', ']', '…', 8) AS matched
               FROM local_file_fts fts
               JOIN local_files f ON f.rowid = fts.rowid
               WHERE local_file_fts MATCH ?1 AND f.status='ACTIVE'
               LIMIT ?2";
    // FTS5 trigram이라 query는 그대로 사용 가능(부분일치)
    let items = db.query_search(sql, &query, limit)?;
    Ok(LocalFileSearchResult { items })
}
```

### 5.4 서버 반영 (outbox → 백엔드)

`sync_enabled=1`인 폴더의 이벤트만 outbox에 적재하고, `flush_sync_outbox`가 백엔드로 보낸다. 원본 파일이 아니라 **메타·이벤트만**(개인 원문 비업로드). 실제 업로드는 사용자가 "올리기"를 누른 자료만 `POST /api/resources`(multipart)로 간다.

```rust
#[tauri::command]
pub async fn flush_sync_outbox(db: State<'_, Db>, token: String)
  -> Result<SyncOutboxFlushResult>
{
    let pending = db.outbox_pending()?;
    let (mut sent, mut failed) = (0, 0);
    for row in pending {
        let res = http_post(&row.endpoint, &row.payload_json,
                            &row.idempotency_key, &token).await; // /api/local-file-events/sync
        match res { Ok(_) => { db.outbox_mark_sent(&row.outbox_id)?; sent += 1; }
                    Err(_) => { db.outbox_bump_retry(&row.outbox_id)?; failed += 1; } }
    }
    Ok(SyncOutboxFlushResult { sent_count: sent, failed_count: failed, flushed_at: now() })
}
```

```ts
function useResourceSearch(query: string) {
  // 1) 로컬: 타우리 IPC, 즉시
  const local = useQuery({
    queryKey: ["local-search", query],
    queryFn: () => tauriCommands.searchLocalFiles({ query, limit: 30 }),
    enabled: isTauriRuntime() && query.length > 0,
  });
  // 2) 서버: 의미검색 RAG, 뒤이어
  const server = useQuery({
    queryKey: ["semantic-search", query],
    queryFn: () => apiRequest("/api/ai/search-resource", { method: "POST", body: { query } }),
    enabled: query.length > 1,
  });
  // 3) 병합: 로컬 먼저 렌더, 서버 도착 시 출처 합치고 중복 제거
  return mergeResults(local.data, server.data);
}
```

UI 규칙:
- 결과 카드에 **출처**(로컬 폴더 경로 / 서버 자료) + matchedText 스니펫 + 미리보기.
- 모드 토글: 파일명 / 본문 키워드 / 의미. 기본은 통합(내부에서 병합).
- 로컬 결과엔 "내 PC · 안 올라감" 배지, 서버 결과엔 프로젝트룸/공유 배지.
- 관리 폴더 화면(`features/managed-folder`)은 `getIndexProgress`로 색인 진행률 바, watch 상태, "최근 변경 N건 반영" 표시.

`lib/tauri/commands.ts`에 신규 커맨드(`get_index_progress`, `open_local_file`, `set_folder_sync`, `reindex_file`)의 타입을 추가하고, `tauriCommands` 래퍼에 함수를 더한다(기존 `appReady` 패턴 그대로).

## 7. 4주 마일스톤 (착수 순서)

| 주차 | 목표 | 산출물 | 검증 |
|---|---|---|---|
| 1주 | DB·폴더·스캔 | rusqlite 연결, schema.sql, `select/scan_managed_folder`, `local_files` 적재 | 폴더 고르면 파일 목록이 DB에 쌓임 |
| 1주 | 파일명 즉시 검색 | FTS5 name 색인, `search_local_files`(이름) | 검색창에 치면 파일명 결과 즉시 |
| 2주 | 본문 추출 + 본문 검색 | extract(txt/md/docx/pdf), FTS5 content, snippet→matchedText | "계약" 치면 본문 매칭 + 스니펫 |
| 2주 | 한국어 | trigram 토크나이저 확인(계약/계약서/계약금) | 변형어가 걸림 |
| 3주 | 실시간 감시 | notify watcher, `local_file_events`, 재색인, 진행률 emit | 파일 추가/수정/삭제가 자동 반영 |
| 3주 | 서버 의미검색 병합 | `/api/ai/search-resource` 연결, 2계층 병합 UI | 로컬 먼저 + 서버 의미 뒤이어 |
| 4주 | 활동 인식 | `read_activity_context`, 상주 캡처, 동의 게이트 | 동의 후 활동이 버퍼에 쌓임 |
| 4주 | outbox 동기화 | `flush_sync_outbox` → localsync API, idempotency | 동기화 켠 폴더 변경분이 서버 반영 |

시연만(구현 얇게): HWP 추출, OCR, Lindera, 하이브리드 RRF.

## 8. 위험과 대안

| 위험 | 영향 | 대안 |
|---|---|---|
| HWP/HWPX 추출 어려움 | 한국 프리랜서 핵심 형식 | 4주엔 미지원 표기 + 위험 명시. 추후 kordoc(Node 사이드카) 또는 변환 |
| 대용량 폴더 색인 성능 | 발열·지연(Docufinder도 1000건 2~5분) | 앞 1MB 해시, 추출은 백그라운드 큐, FTS 먼저·본문 나중 |
| 한국어 검색 품질 | trigram은 형태소만큼 정밀하지 않음 | 1차 trigram, 품질 이슈 시 Lindera 스트레치 |
| macOS 권한 동의 | 창 제목·폴더 접근 빈 값 | 온보딩 권한 설명 화면, capability 명시 |
| pdf 텍스트 레이어 없음(스캔본) | 추출 실패 | extract_status=UNSUPPORTED, OCR은 서버/보류 |
| DB 동시 접근 | 잠금 오류 | rusqlite 단일 소유 + WAL 모드, 프론트는 IPC만 |
